# EvoFlow 使用说明

> **适用读者**：终端用户、团队管理员  
> **前置条件**：已安装 EvoPanel，并持有至少一个 LLM 服务商 API Key  
> **配套文档**：[产品说明书](./产品说明书.md) · [操作指南目录](./guides/README.md)  
> **桌面端版本**：EvoPanel v0.2.3

## 2. 安装与首次启动

### 2.1 下载安装

1. 打开 [GitHub Releases](https://github.com/EvovexAI/EvoFlow/releases)，下载对应系统安装包：
   - Windows：`EvoPanel_x.x.x_x64-setup.exe`
   - macOS / Linux：按页面提供的对应包安装
2. 按安装向导完成安装并启动 **EvoPanel**。

### 2.2 首次启动

1. **访问密码**（若团队已配置）：按提示输入面板访问密码后进入。
2. **后端就绪**：启动后若出现「后端未启动」，请稍候或点击「重新检测」；桌面版会自动拉起本地后端。
3. **首次配置模型**（必做）：若尚未配置模型，应用会跳转至模型页并提示「欢迎使用 EvoFlow! 请先配置至少一个 AI 模型」。

### 2.3 配置第一个模型

1. 点击左下角 **设置**，在弹窗中选择 **模型** 标签（或直接进入 `#/models`）。
2. 点击 **添加模型**：
   - **厂商**：选择 AI 服务商（OpenAI、Anthropic、DeepSeek、通义等）
   - **模型名称**：选择或填写模型 ID
   - **API 密钥**：填入服务商官网获取的 Key
3. 点击 **测试连接**，成功后勾选 **设为默认模型** 并保存。

### 2.4 发送第一条消息

1. 回到 **新建对话**（`#/chat`）。
2. 在输入框输入需求，例如：`帮我列出当前工作区里的 Markdown 文件`。
3. 按 Enter 发送，观察流式回复与工具调用过程。

## 4. 基础对话

### 4.1 发送消息

1. 在输入框输入文字，可附加 **图片**（上传按钮）。
2. 输入 `@` 可引用 **工作区** 内文件作为上下文。
3. 点击发送或 Enter；生成中可点击 **停止生成**。

### 4.2 会话管理

- **新建对话**：左侧「新建对话」或指令 `/new`（新线程，上下文隔离）。
- **切换会话**：在会话列表点击历史项。
- **右键菜单**：置顶、重命名、删除会话。

### 4.3 工具调用与审批

Agent 执行工具时，对话中会显示对应块（如「终端命令」「读取文件」「子智能体」等）。

涉及副作用操作时可能出现审批条：

| 操作 | 说明 |
|------|------|
| **批准并执行** | 允许本次调用 |
| **拒绝** | 阻止本次调用 |
| **全部授权** | 当前会话内同类操作自动批准 |

在 **设置 → 通用** 中可开启「新对话默认不询问」，自动批准终端、删文件等（请仅在可信环境使用）。

### 4.4 通用设置建议

**设置 → 通用** 中常用项：

- **外观**：日间 / 夜间 / 跟随系统
- **记忆与上下文**：默认启用记忆（跨会话积累偏好与事实）
- **文件系统模式**：启用虚拟沙箱路径 `/mnt/user-data/...`
- **用户工作空间**：指定数据根目录

## 5. 对话模式与场景

### 5.1 四种模式对照

在 Composer 的 **模式** 控制项中切换（或与场景联动）：

| 模式 | 适用场景 |
|------|----------|
| **Chat** | 快速问答、轻量任务 |
| **Plan** | 先出完整方案，确认后再执行 |
| **Execute** | 需要子 Agent 并行执行的任务 |
| **Infinite** | 复杂、多步骤长任务 |

### 5.2 Plan 模式完整流程

适用于多步骤开发、研究分析等：

1. 发送复杂需求（或切换到 **任务规划** 场景）。
2. AI 进入 **规划中**，输出任务拆解、步骤与预期结果。
3. 审阅方案：不满意可直接回复修改要求。
4. 确认后点击 **开始执行**（或回复「确认执行」）。
5. 进入 **执行中**：可在侧栏查看进度；可随时暂停、补充指令。

侧栏常见状态：**规划中** → **待授权开始执行** → **执行中** → **校验中**。

> 详细机制见 [Plan 模式指南](./guides/plan-mode.md)。

### 5.3 工作区场景

1. 切换到 **工作区** 场景。
2. 使用 **工作区文件** 查看、预览项目内文件。
3. 在对话中用 `@文件名` 将文件注入上下文。

## 6. 任务中心

适用于竞品分析、系统开发等需 **多 Agent 协作** 的大型任务。

### 6.1 创建任务

1. 左侧点击 **任务中心**。
2. 右上角 **新建任务**，填写：
   - **任务名称**：简短标题
   - **任务描述**：需求、输出格式、约束等（越详细拆解越准）
3. 确定后 Supervisor 自动进入 **规划中**，拆分子任务。

### 6.2 任务状态

| 状态 | 说明 |
|------|------|
| 待开始 | 已创建，未开始规划 |
| 规划中 | 正在拆解子任务 |
| 执行中 | 子任务运行中 |
| 已暂停 | 可恢复 |
| 已完成 | 全部子任务完成 |
| 失败 / 已取消 | 异常或人工取消 |

### 6.3 常用操作

| 操作 | 时机 |
|------|------|
| **开始执行** | 规划完成、确认计划后 |
| **暂停** | 执行中需人工介入 |
| **查看详情** | 查看子任务列表与进度 |
| **实时对话** | 跳转聊天页与执行 Agent 交互 |
| **批量选择** | 批量启动 / 暂停 / 删除 |

### 6.4 任务详情页

- **任务计划**：查看 AI 生成的完整拆解文档。
- **子任务卡片**：单独暂停、重试、查看结果。
- **对话记录**：右侧展示调度与工具调用日志。

> 详见 [任务中心操作指南](./guides/task-center.md)。

## 7. 托管任务

托管任务在**后台多轮自动执行**，最长支持 7×24 小时，适合巡检、监控、周期性处理等。

### 7.1 创建方式

**方式一：界面（推荐）**

1. 在聊天页 Composer 点击 **托管**。
2. 右侧面板填写：
   - **任务目标**：描述要自动完成的事
   - **总轮次**：执行次数；`0` 表示不限次数直至手动停止
   - **自动停止时间**（可选）：分钟数，最长 10080（7 天）
   - **结果推送**：开启后推送到已配置的通知渠道
3. 点击 **启动托管**。

**方式二：自然语言**

在对话中直接描述，例如：

```
帮我创建一个托管任务，每小时检查一次服务器状态，运行 7 天后自动停止
```

**方式三：斜杠指令**

```
/hosted 每小时检查服务器磁盘占用，超过 90% 时告警
```

### 7.2 运行中管理

右侧面板实时显示：状态、已执行轮次、最近结果。

| 操作 | 说明 |
|------|------|
| **暂停 / 恢复** | 临时停止或继续 |
| **终止** | 结束托管 |
| **聊天补充** | 运行中可直接发消息调整目标或逻辑，无需重建 |

### 7.3 模型提议托管

助手可能返回托管方案并在聊天区显示 **确认条**：确认后才会真正启动；也可仅填入面板、稍后手动启动。

### 7.4 飞书结果推送

配置飞书渠道后，托管结束可向飞书线程推送 **Markdown 结果汇总**。详见第 12 章。

> 详见 [托管智能体操作指南](./guides/hosted-agent.md)。

## 8. 定时任务

### 8.1 页面创建

1. 左侧 **定时任务** 进入 `#/cron`。
2. 右上角 **新建任务**，配置：

| 配置项 | 说明 |
|--------|------|
| 任务名称 | 便于识别的标题 |
| 任务描述（Prompt） | 每次触发时 Agent 要完成的指令 |
| 调度模式 | 每小时 / 每天 / 每周 / 一次性 / 自定义 Cron |
| 飞书推送 | 可选，填写 chat_id |
| 记忆 | 开启后多次执行可复用上下文 |

3. 点击 **创建任务** 启用。

**Cron 示例**：`0 9 * * 1-5` 表示周一至周五 9:00。

### 8.2 对话式管理

在聊天中描述即可，例如：

```
帮我创建一个定时任务，每天早上 9 点生成昨日业务数据报表，结果推送到飞书群
```

或使用 `/automation` 指令（见附录）。

### 8.3 任务列表操作

| 按钮 | 功能 |
|------|------|
| ▶ 运行 | 手动触发一次 |
| ⏸ 暂停 / ↻ 恢复 | 控制调度 |
| 📋 历史 | 查看执行记录 |
| ✎ 编辑 / ✕ 删除 | 修改或移除 |

> 详见 [定时任务与通知推送指南](./guides/scheduled-tasks.md)。

## 9. 角色（Agent）管理


### 9.3 切换角色

- 界面：Composer → **角色**

## 12. IM 渠道（团队管理员）

### 12.1 飞书机器人配置

1. 登录 [飞书开放平台](https://open.feishu.cn/)，创建**企业自建应用**。
2. 获取 **App ID**、**App Secret**；开启机器人能力。
3. 按 EvoPanel 提示配置**事件订阅**与**消息回调地址**。
4. EvoPanel：**设置 → IM 通信 → 飞书**，填入凭证并**开启飞书渠道**。
5. 在开放平台**发布应用**（企业内部可用即可）。

### 12.2 会话映射

1. **设置 → IM 通信 → 飞书 → 会话映射**。
2. **添加映射**：选择飞书群/用户，绑定指定 **Agent**。
3. 未映射时，用户可通过 `/agent` 动态切换。

### 12.3 飞书端常用能力

- 直接发消息对话（流式卡片更新）
- 斜杠指令（见附录）：`/new`、`/hosted`、`/task`、`/automation` 等
- 托管进度与结果自动推送
- 群聊需 **@机器人** 后再输入指令，例如：`@EvoFlow /status`

### 12.4 其他渠道

Slack、Telegram、钉钉、Discord 等同理，在 **设置 → IM 通信** 中配置。详见 [IM 消息渠道配置](./guides/im-channels.md)。

## 15. 常见问题与排查

### 15.1 后端未启动 / Gateway 未就绪

| 现象 | 处理 |
|------|------|
| 全屏「后端未启动」 | 等待自动拉起；Tauri 版查看日志；点击「重新检测」 |
| 聊天提示 Gateway 未就绪 | 确认本地后端进程正常；桌面版端口见 `~/.evoflow/evopanel/backend-runtime.json`（Windows：`%USERPROFILE%\.evoflow\evopanel\`） |

### 15.2 模型连接失败

1. 检查 API Key 是否正确、未过期。
2. **测试连接** 失败时查看错误信息（余额、模型 ID、网络代理）。
3. 自定义网关需填写正确的 **Base URL**。

### 15.3 修改 Agent 配置不生效

部分配置需 **新建会话** 或重置 Agent 后生效。

### 15.4 定时任务不执行

确认调度器已启动（Gateway 默认开启；环境变量 `EVOFLOW_AUTOMATION_SCHEDULER=0` 会关闭）。

### 15.5 飞书无回复

1. 确认应用已发布、机器人已加入群聊。
2. 检查回调地址与事件订阅是否与 EvoPanel 一致。
3. 群聊是否 @了机器人。

### 15.6 数据备份

EvoPanel 本地数据目录：`~/.evoflow/`（含任务配置、设置备份）。定期备份该目录即可。

> 更多问题见 [FAQ](./faq.md)。
