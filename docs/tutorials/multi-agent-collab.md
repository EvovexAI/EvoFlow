## 步骤

### 1. 理解多智能体协作

多智能体协作（Project）是 EvoFlow 中处理复杂项目的高级机制。它的核心思想是：

- **Supervisor（超级总控智能体）**：负责任务拆解、分配和协调的中枢
- **Sub-agent（子代理）**：执行具体任务的专项 Agent，每个可以有不同的角色、技能和模型
- **任务编排**：支持并行和串行两种执行策略

适用场景包括：研究报告撰写、代码审查与修复、数据分析流水线、内容创作等需要多个专业角色协作的工作。

#### 协作架构

```
Supervisor（超级总控智能体）
  |---> Sub-agent 1: 信息搜集
  |---> Sub-agent 2: 数据分析
  |---> Sub-agent 3: 报告撰写
  |---> Sub-agent 4: 质量审核
```

超级总控智能体接收用户指令，将大任务拆解为子任务，分派给不同的子代理，最后汇总各子代理的结果，产出最终交付物。

### 2. 创建项目：以「行业研究报告」为例

假设你要完成一份行业研究报告，这个任务涉及多个环节。我们来创建一个多 Agent 协作项目。

#### 2.1 定义项目结构

一个完整的项目包含：

```
项目名称: AI 行业研究报告
  |
  +-- Supervisor（超级总控智能体）: 项目经理（负责整体协调）
  |
  +-- Sub-agent 1: 研究员（负责信息搜集）
  +-- Sub-agent 2: 分析师（负责数据分析）
  +-- Sub-agent 3: 写手（负责报告撰写）
  +-- Sub-agent 4: 审核员（负责质量审核）
```

#### 2.2 在 EvoPanel 中创建项目

1. 打开 EvoPanel，进入 **项目管理** 页面
2. 点击 **新建项目**
3. 填写项目信息：

| 字段 | 值 |
|------|------|
| 项目名称 | `AI-industry-report` |
| 项目描述 | 生成一份 2026 年 AI 行业研究报告 |
| Supervisor Agent | 选择默认的 `supervisor` 超级总控智能体 |

### 3. 添加子代理

在项目的 **Agent 配置** 页面中添加子代理：

#### 研究员

| 字段 | 值 |
|------|------|
| 名称 | `researcher` |
| 角色 | `researcher` |
| SOUL | `你是一个专业的信息研究员，擅长从各种渠道搜集、整理和验证信息。你的工作是为分析团队提供准确、全面的原始素材。` |
| 模型 | 选择搜索能力强的模型 |

#### 分析师

| 字段 | 值 |
|------|------|
| 名称 | `analyst` |
| 角色 | `analyst` |
| SOUL | `你是一个资深数据分析师，擅长从原始数据中发现趋势、模式和洞察。你的分析应该数据驱动、客观准确。` |
| 模型 | 选择推理能力强的模型 |

#### 写手

| 字段 | 值 |
|------|------|
| 名称 | `writer` |
| 角色 | `writer` |
| SOUL | `你是一个专业的科技写作者，擅长将复杂的技术内容转化为结构清晰、语言流畅的报告。你的文章结构严谨、引用规范。` |
| 模型 | 选择写作能力强的模型 |

#### 审核员

| 字段 | 值 |
|------|------|
| 名称 | `reviewer` |
| 角色 | `reviewer` |
| SOUL | `你是一个严谨的质量审核员，负责检查报告的准确性、完整性和一致性。你会指出事实错误、逻辑漏洞和格式问题。` |
| 模型 | 选择推理能力强的模型 |

### 4. 任务规划与编排

#### 4.1 启动项目

在项目的 **任务** 面板中，输入你的主任务指令：

```
请完成一份 2026 年 AI 行业研究报告，包含以下内容：
1. 行业发展概况
2. 关键技术趋势分析
3. 主要玩家与竞争格局
4. 市场规模与预测
5. 风险与机遇分析

报告要求：数据准确、引用来源、不少于 3000 字。
```

#### 4.2 超级总控智能体自动拆解

超级总控智能体接收到任务后，会自动进行任务拆解：

1. **任务 1**（并行）：研究员搜集 AI 行业最新信息、政策、融资事件
2. **任务 2**（并行）：分析师搜集市场规模数据、增长率预测、投融资趋势
3. **任务 3**（串行，依赖 1、2）：写手基于研究员和分析师的材料撰写报告初稿
4. **任务 4**（串行，依赖 3）：审核员检查报告质量，提出修改意见

#### 4.3 执行策略

| 阶段 | 子任务 | 执行方式 | 说明 |
|------|--------|---------|------|
| 信息搜集 | Task 1 + Task 2 | 并行 | 两个 Agent 同时工作 |
| 报告撰写 | Task 3 | 串行 | 等待前两个任务完成后开始 |
| 质量审核 | Task 4 | 串行 | 等待报告完成后开始 |

超级总控智能体会根据任务间的依赖关系自动选择最优执行策略。

### 5. Agent 协调机制

#### 5.1 信息传递

子代理之间通过超级总控智能体进行信息传递：

```
研究员 ──(结果)──> Supervisor ──(素材)──> 写手
分析师 ──(结果)──> Supervisor ──(素材)──> 写手
                                      │
                                  (报告初稿)
                                      │
                                      v
审核员 <── Supervisor <── 写手
```

#### 5.2 协调规则

- **Push-based（推送式）**：子代理完成后，结果自动推送给超级总控智能体，无需轮询
- **依赖管理**：超级总控智能体确保依赖任务完成后才触发后续任务
- **深度限制**：默认子代理嵌套深度最大为 3 层，防止无限递归
- **数量限制**：每个 Agent 同时最多 5 个子代理

#### 5.3 结果汇总

所有子任务完成后，超级总控智能体将各阶段结果进行汇总，生成最终交付物返回给用户。

### 6. 在 EvoPanel 中监控进度

#### 6.1 项目总览

打开 EvoPanel 的 **项目管理** 页面，可以看到：

- 项目当前状态（执行中/已完成/失败）
- 总体进度百分比
- 各子任务的状态（待执行/执行中/已完成/失败）

#### 6.2 子任务详情

点击任一子任务，可以查看：

- 分配给哪个 Agent
- 当前执行状态
- 实时输出流（SSE 流式渲染）
- 已完成任务的输出结果
- 运行时间与 Token 消耗

#### 6.3 日志与历史

在项目页面底部，可以查看：

- 完整的执行日志
- 各子任务的输入和输出
- 超级总控智能体的拆解和调度记录
- 历史执行记录（每次运行的快照）

### 7. 实际示例：执行并观察

完成上述配置后，点击 **执行项目**，你将看到：

```
[超级总控智能体] 收到任务: 完成 AI 行业研究报告
[超级总控智能体] 拆解为 4 个子任务

[Task 1] 研究员开始执行: 搜集 AI 行业信息...
[Task 2] 分析师开始执行: 搜集市场规模数据...
         ↑ 两个任务并行执行

[Task 1] 研究员完成，输出 5000 字素材
[Task 2] 分析师完成，输出数据图表 12 份

[Task 3] 写手开始执行: 基于素材撰写报告...

[Task 3] 写手完成，输出报告初稿 3500 字

[Task 4] 审核员开始执行: 质量审核...

[Task 4] 审核员完成，发现 2 处需要修改

[超级总控智能体] 所有子任务完成，汇总输出最终报告
```

## 你完成了！

你已成功创建并执行了一个多智能体协作项目。超级总控智能体将大任务拆解为子任务，多个 Agent 并行和串行协作，最终产出了一份完整的研究报告。
