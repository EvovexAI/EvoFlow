# 常见问题与排查指南

> 本文档分为两部分：**用户使用排查**（普通用户场景）和**部署与开发**（开发者/运维场景）。

---

# 一、用户使用排查

## 1. 模型连接相关问题

### 问题：提示模型连接失败

排查步骤：

1. 检查 API 密钥是否正确，有没有多余空格、换行符等特殊字符
2. 确认 API 密钥有对应模型的调用权限，账户余额/配额充足
3. 检查接口地址（base_url）是否正确，有没有多写或少写路径后缀（比如不要漏了 `/v1`）
4. 确认本地网络可以正常访问模型服务商的接口，可以尝试在终端 ping 或 curl 测试
5. 如果是代理访问，检查代理配置是否正确，代理服务器是否正常运行
6. 查看 `logs/gateway.log` 中的详细错误日志，确认是网络问题还是鉴权问题

### 问题：模型回复慢/超时

排查：

1. 检查网络连接是否正常，是否有网络波动
2. 确认模型服务商的服务状态，是否有服务降级或故障
3. 尝试降低最大生成长度参数，减少输出内容
4. 如果是复杂任务，可以拆分成多个小任务分别处理
5. 切换到其他模型尝试，确认是否是特定模型的问题

---

## 2. 工具/技能调用问题

### 问题：AI 不会调用技能/工具

排查：

1. 进入「技能」页面，确认该技能已经安装并处于启用状态
2. 进入「智能体」页面，编辑当前使用的智能体，确认已经勾选了对应技能/工具的权限
3. 需求描述尽量明确，明确说明要使用什么技能/工具做什么事，比如「使用 Python 运行工具执行这个脚本」
4. 检查模型是否支持工具调用，部分轻量化模型可能不支持工具调用能力

### 问题：工具执行失败

排查：

1. 检查本地权限是否足够，比如终端执行命令需要对应文件/命令的操作权限，写入文件需要目录写入权限
2. 确认依赖环境已经安装，比如运行 Python 脚本需要本地有 Python 环境，相关依赖包已经安装
3. 点击工具调用卡片的「查看日志」按钮，查看详细错误信息
4. 如果是 MCP 工具，检查 MCP 服务器是否正常运行，配置是否正确
5. 尝试手动在本地执行相同命令，确认命令本身是否可以正常运行

### 问题：MCP 服务器连接失败

排查：

1. 进入「工具」页面，检查 MCP 服务器是否处于启用状态
2. 确认 MCP 服务器的配置参数正确：命令路径、参数、URL 地址等
3. 如果是 stdio 类型的 MCP，确认命令依赖的运行环境已经安装
4. 如果是 SSE/HTTP 类型的 MCP，确认服务已经启动，网络可以正常访问对应地址
5. 查看 `logs/gateway.log` 中的 MCP 相关错误日志

---

## 3. 托管/定时任务问题

### 问题：托管任务运行失败

排查：

1. 点击托管任务面板的「查看日志」，查看具体的错误信息
2. 确认模型配置正确、API 配额充足，没有调用限制
3. 确认任务目标描述清晰，没有歧义，如果需要特定上下文或权限请补充说明
4. 检查是否需要用户补充信息，任务是否处于等待反馈状态
5. 查看 `logs/hosted.log` 中的托管任务相关日志

### 问题：托管任务自动停止

排查：

1. 检查是否达到了设置的最大运行轮次或自动停止时间
2. 查看是否是任务已经完成，达到了停止条件自动结束
3. 检查是否连续执行失败超过了重试阈值，自动暂停
4. 确认 EvoFlow 客户端是否处于运行状态，没有被关闭

### 问题：定时任务没触发

排查：

1. 进入「定时任务」页面，确认定时任务状态是「已启用」
2. 检查 Cron 表达式是否正确，可以使用在线 Cron 表达式验证工具确认
3. 确认 EvoFlow 客户端处于运行状态，没有被关闭，电脑没有休眠
4. 查看定时任务的执行历史，是否有执行失败记录
5. 查看 `logs/automation.log` 中的定时任务相关日志
6. 如果是飞书推送任务，确认飞书渠道配置正确，机器人有发送消息权限

---

## 4. 客户端运行问题

### 问题：EvoFlow 启动闪退/打不开

排查：

1. 检查是否有其他进程占用了默认端口 8012、2024，可以使用 `netstat -ano` 命令查看端口占用
2. 尝试以管理员/root 权限运行，避免权限不足导致的启动失败
3. 查看安装目录下 `logs` 文件夹的错误日志
4. 尝试删除 `~/.evoflow/config.yaml` 配置文件，重新启动（会重置配置，建议先备份）
5. 卸载重装最新版本，确认下载的安装包对应你的操作系统版本

### 问题：聊天页面加载异常/空白

排查：

1. 刷新页面重试，或按 Ctrl+F5 强制刷新
2. 检查本地网络是否正常，是否可以正常访问其他网站
3. 重启 EvoFlow 客户端尝试
4. 清除浏览器缓存（如果是 Web 端使用）
5. 查看控制台错误信息（按 F12 打开开发者工具），确认具体错误原因

### 问题：消息发送没反应

排查：

1. 检查是否有正在运行中的任务，等待当前任务完成后再发送
2. 确认模型配置正确，没有连接失败
3. 检查输入内容是否为空，或包含违规内容
4. 重启 EvoFlow 客户端尝试
5. 查看 `logs/gateway.log` 中的错误日志

---

## 5. 飞书渠道相关问题

### 问题：飞书发消息没有回复

排查：

1. 确认飞书应用配置正确，事件订阅、消息接收权限、机器人能力都已经开启
2. 确认回调地址是公网可访问的地址，不能是内网地址或 localhost
3. 进入 EvoFlow「设置」→「飞书配置」，确认飞书渠道已经开启，App ID 和 App Secret 填写正确
4. 确认消息内容不是敏感词被飞书拦截，尝试发送简单的测试消息如「你好」
5. 查看 `logs/feishu.log` 中的飞书相关日志
6. 确认机器人在群聊中有 @ 权限，或者单聊时已经添加机器人为好友

### 问题：收不到推送通知

排查：

1. 进入「设置」→「通知配置」，确认推送渠道配置正确，点击「测试推送」可以收到测试消息
2. 确认任务的推送规则已经开启对应通知类型（任务完成、异常告警、进度更新等）
3. 检查飞书机器人有没有被禁言，有没有在对应群聊/单聊的发消息权限
4. 确认飞书账号没有设置免打扰，消息没有被归类到消息盒子
5. 查看 `logs/feishu.log` 中的推送相关日志，确认推送是否成功

### 问题：飞书卡片内容显示不全/乱码

排查：

1. 这通常是因为内容过长导致，飞书卡片有字数限制，过长内容会自动截断
2. 可以点击卡片的「查看完整内容」链接查看完整内容
3. 如果是乱码，确认飞书客户端是最新版本，尝试升级客户端
4. 如果是代码块显示异常，可以复制内容到本地编辑器查看

# 二、部署与开发

## 安装与启动

### 运行 `make dev` 前需要什么？

确保已安装以下依赖：

- Python 3.12+（后端）
- Node.js 18+（EvoFlow 前端）
- uv（Python 包管理器）

首次运行建议先执行 `make install` 安装所有依赖。

### `make dev` 和 `make start` 有什么区别？

| 命令 | 模式 | 热重载 | 适用场景 |
|------|------|--------|----------|
| `make dev` | 开发模式 | 是 | 日常开发 |
| `make start` | 生产模式 | 否 | 部署测试 |

### 启动后端口被占用怎么办？

EvoFlow 使用以下端口（开发脚本 / 桌面内置 sidecar 默认；Docker 部署可能使用 8001 等，以 `config.yaml` 为准）：

| 端口 | 服务 |
|------|------|
| 2024 | LangGraph Server（桌面 sidecar 默认，可在 2024–2062 扫描） |
| 8012 | Gateway API（桌面 sidecar 默认，可在 8012–8062 扫描） |
| 8001 | Gateway API（`make gateway` / Docker 常见） |
| 1420 | EvoFlow 前端开发（Vite） |
| 8002 | Provisioner（可选） |

桌面版实际端口写入 `~/.evoflow/evopanel/backend-runtime.json`；若 8012 被占用会自动改用同段其它端口。排查「网关不可用」时可对照该文件中的 `port` / `baseUrl`，或设置环境变量 `EVOFLOW_GATEWAY_URL` 指向健康实例。

运行 `make stop` 停止所有服务后重试。

### 如何在后台运行？

使用 `make dev-daemon` 在后台启动所有服务。使用 `make stop` 停止。

## 模型配置

### 支持哪些模型？

EvoFlow 支持 OpenAI、Anthropic Claude、Google Gemini、DeepSeek、阿里 DashScope、Moonshot (Kimi)、MiniMax、火山引擎等主流模型。在 `config.yaml` 的 `models` 列表中配置。

### 如何切换模型？

在 EvoFlow 界面中选择模型，或通过运行时参数 `model_name` 指定。

### thinking 模式是什么？

某些模型（如 Claude）支持扩展思考模式（`thinking_enabled: true`），让模型在回答前进行深度推理。这会增加响应时间但提高质量。

### vision 模式是什么？

当模型支持视觉（`supports_vision: true`）时，Agent 可以使用 `view_image` 工具查看图片。这需要在 `config.yaml` 中正确标记模型能力。

## Docker 与沙箱

### 本地沙箱和 Docker 沙箱有什么区别？

| 模式 | 隔离级别 | 适用场景 |
|------|----------|----------|
| 本地 | 仅路径隔离 | 单用户、可信环境 |
| Docker | 容器隔离 | 开发、测试、多用户 |
| Kubernetes | Pod 隔离 | 生产环境 |

### 沙箱容器镜像下载很慢怎么办？

运行 `make setup-sandbox` 预拉取沙箱容器镜像。这会在启动前提前下载镜像，避免首次使用时的等待时间。

### 容器沙箱最大并发数是多少？

默认 3。达到上限时，最久未使用的容器会被 LRU 淘汰。

## IM 消息渠道

### 对接飞书需要什么？

在 `config.yaml` 中配置飞书渠道，设置以下环境变量：

- `FEISHU_APP_ID`：飞书应用 ID
- `FEISHU_APP_SECRET`：飞书应用密钥

飞书渠道通过 outbound 连接（WebSocket），无需公网 IP。

### 支持哪些 IM 平台？

飞书、钉钉、Telegram、Slack、Discord。

### 如何同时支持多个渠道？

在 `config.yaml` 的 `channels` 中配置多个渠道，每个渠道独立配置 App ID 和密钥。

## 记忆系统

### 记忆存储在哪里？

全局记忆存储在 `backend/.evo-flow/memory.json`，按线程隔离的记忆存储在各 `threads/{thread_id}/` 目录下。

### 如何清除记忆？

直接删除或编辑 `backend/.evo-flow/memory.json` 文件，重启服务后生效。

### Agent 能记住上一次对话吗？

可以。记忆系统会自动从对话中提取关键事实并持久化。下次对话时，相关记忆会注入系统提示中。

## 技能与 Agent

### 如何添加自定义技能？

1. 在 `skills/custom/` 下创建目录
2. 在目录中创建 `SKILL.md` 文件
3. 在 `extensions_config.json` 中启用

或通过 API 安装 `.skill` 压缩包。

### 如何创建自定义 Agent？

通过 API 创建：

```bash
curl -X POST http://localhost:8001/api/agents \
  -H "Content-Type: application/json" \
  -d '{"agent_code": "my-agent", "agent_name": "我的Agent", "description": "..."}'
```

创建后在 `agents/my-agent/` 目录下配置 `config.yaml` 和 `SOUL.md`。

### 子 Agent 是什么？

子 Agent 是主 Agent 委派任务的机制。主 Agent 可以将任务拆分，委派给多个子 Agent 并行执行，最后综合结果。

## 自动化调度

### 如何创建定时任务？

在自动化调度器中配置 cron 或 rrule 表达式，指定触发时间和任务内容。支持飞书推送通知。

### 调度器默认启用吗？

默认启用。设置 `EVOFLOW_AUTOMATION_SCHEDULER=0` 可禁用。

## 其他

### `config.yaml` 和 `extensions_config.json` 有什么区别？

- `config.yaml`：主配置文件，定义模型、沙箱、工具、渠道等
- `extensions_config.json`：扩展配置文件，控制 MCP 服务器和技能的启用/禁用状态

### 对话里如何用 `@@路径@@` 直接点开文件？

助手在回复正文里用 **首尾各一对 `@@`** 包住相对路径，EvoFlow 会渲染成可点击链接，点击后与工作区文件树相同，在弹窗中预览。未闭合的 `@@outputs/...` 或单个 `@` **不会**识别。

约定示例：

- `@@outputs/2026-05-AI日报.html@@`
- `@@workspace/third_sample_document.md@@`

路径须在本会话工作区的 `outputs/` 或 `workspace/` 下；不要用 Windows 绝对路径或 http 外链代替。写文件仍用 `write_to_file`；在回复中用 `@@路径@@` 交付即可，EvoFlow 会识别为可点击链接。

### 如何查看 API 文档？

启动服务后访问 `http://localhost:8001/docs` 查看 FastAPI 自动生成的 Swagger UI。
