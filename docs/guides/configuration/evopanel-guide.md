# EvoPanel 桌面端使用指南

## 适用场景
通过 EvoPanel 桌面客户端与 EvoFlow 交互，提供完整的 GUI 界面来管理对话、Agent、任务和集成。

## 前置条件
- EvoFlow 后端已运行（localhost:2026）
- 已安装 EvoPanel（Tauri v2 桌面应用）

## 什么是 EvoPanel

EvoPanel 是基于 Tauri v2 构建的桌面客户端，为 EvoFlow 提供图形化交互界面。包含 AI 对话、Agent 管理、任务中心、多代理协作、技能与 IM 管理等完整功能。

## AI 助手

### 四种对话模式

| 模式 | thinking_enabled | is_plan_mode | subagent_enabled | 适用场景 |
|------|-------------------|--------------|-------------------|----------|
| Chat | false | false | false | 快速问答 |
| Plan | true | true | false | 规划分析 |
| Execute | true | false | true | 任务执行 |
| Infinite | true | true | true | 复杂多步骤任务 |

### 实时聊天

- 流式响应，逐字显示 AI 回复
- Markdown 渲染（代码高亮、表格、数学公式）
- 多模态支持（图片理解、文件上传）
- 工具调用过程可视化

## 模型配置

通过模型配置 UI 可：
- 选择当前使用的模型
- 配置模型的 thinking/vision 开关
- 添加新的模型提供商（OpenAI、Anthropic、Google、DeepSeek 等）
- 设置基础 URL 和 API Key

## 任务中心

- **创建任务**：从模板或自定义创建自动化任务
- **监控执行**：实时查看任务执行状态和输出
- **批量操作**：批量启动/暂停/删除任务
- **执行历史**：查看过往执行记录和结果

## 多代理协作

### 项目管理

- 创建项目，组织多个 Agent 协同工作
- 配置 Supervisor 超级总控智能体（主管 Agent），负责任务分配和协调
- 每个项目有独立的会话和上下文

### Agent 管理

- **SOUL/IDENTITY 编辑**：自定义 Agent 的人格和身份描述
- **模型配置**：为不同 Agent 分配不同的模型
- **工具集配置**：控制每个 Agent 可用的工具
- **记忆隔离**：每个 Agent 有独立的记忆文件

## 工具与技能管理

- 浏览已加载的技能（`skills/public/` 和 `skills/custom/`）
- 启用/禁用单个技能
- 从 `.skill` 归档安装新技能
- 管理 MCP 服务器的启用状态

## IM 渠道管理

- 配置飞书、钉钉、Telegram、Slack、Discord 等消息渠道
- 查看各渠道连接状态
- 管理渠道会话和线程映射

## 托管（Hosted）

- 在**实时聊天**输入栏使用 **[托管](https://www.evovexai.com/docs/chat/hosted)** 入口（非「记忆」页）：配置托管参数、启动与停止后台运行。
- **模型提议托管**：助手可返回托管方案，聊天区出现**确认条**——确认后才会开始执行（也可只把方案填入面板、稍后在面板里再点启动）。
- **与飞书联动**：托管结束后可向飞书线程推送 **Markdown 结果汇总**（需网关与渠道允许推送）；在飞书中发送 **「开始」「确认」** 等，可在 EvoPanel **在线且已连接网关**时触发桌面侧应用托管方案并开始（详见官网托管文档中的 IM 与桌面协同说明）。

## 定时任务

- 创建 cron 调度的自动化任务
- 配置 LangGraph 执行和飞书推送
- 选择线程模式（fresh/sticky）

## 记忆管理

- 查看全局和按 Agent 分类的记忆数据
- 手动编辑或删除特定事实
- 导入/导出 `memory.json` 文件
- 配置外部记忆插件

## 主题与本地化

- 支持亮色/暗色主题切换
- 多语言界面（中文、英文、日语、韩语、法语、德语、西班牙语、葡萄牙语、俄语、越南语等）

## 验证是否生效

1. 启动 EvoPanel 应用
2. 确认连接状态指示器显示已连接到 EvoFlow（localhost:2026）
3. 发送一条测试消息，确认 AI 正常响应

## 常见问题

**Q: EvoPanel 无法连接到 EvoFlow？**
确认后端已运行（`make dev` 或 `make docker-start`）。检查连接设置中的 URL 是否正确。

**Q: 修改了 Agent 配置后不生效？**
部分配置修改后需要重置 Agent 或新建会话才能生效。

**Q: 定时任务不执行？**
确认定时任务引擎已启动。Gateway 默认运行调度循环，可通过 `EVOFLOW_AUTOMATION_SCHEDULER=0` 关闭。

**Q: 如何备份 EvoPanel 数据？**
EvoPanel 数据存储在 `~/.evoflow/` 目录下。备份该目录即可保留所有任务配置和本地设置。

# 菜单与面板速查表

EvoPanel 桌面客户端的功能入口分为三层：**左侧主导航**（全屏管理页）、**设置弹窗**（轻量配置 Tab）、以及**隐藏路由**（由列表/任务卡片等跳转进入的二级页面）。本章按层级列出全部入口，每项给出定位语与能力要点，供查阅与培训使用。

## 一、左侧主导航

> 路由表来源：`evopanel/src/components/sidebar.js` 中的 `NAV_ITEMS_FULL`。

### 1.1 概览

#### 实时聊天 `/chat`

与主智能体进行流式对话的统一入口，承载 EvoFlow 几乎所有任务发起场景。

- 实时流式输出，工具调用、子智能体执行、思考过程均可在消息流中即时可视化
- 支持 Chat / Plan / Execute / Hosted（托管）等会话模式切换，复杂任务可在「任务规划」与「执行」阶段间无缝衔接
- 集成图片、文件上传与 `@文件` 工作区引用；Composer 支持模型、角色、记忆、工作空间等运行时控件
- 顶栏「场景」选择决定工具画像（日常对话、工作区、网络搜索、创意媒体等），同一会话可按需切换

### 1.2 配置

#### 模型配置 `/models`

集中管理对话与多模态模型的服务商凭证、能力标签与默认选择。

- 一键添加 OpenAI、Anthropic、Google、DeepSeek、通义、Moonshot、火山等主流厂商，或自定义 OpenAI 兼容网关
- 「测试连接」实时校验密钥与可达性；支持设默认对话模型与默认创意媒体模型
- 标注 `thinking_enabled`、`supports_vision` 等能力位，决定 Agent 能否使用扩展思考与视觉工具
- 支持多实例切换：本机、Docker、远程实例之间的模型清单各自独立

#### Agent 管理 `/agents`

预设角色（智能体）的工坊：维护 SOUL、系统提示词、能力白名单与项目团队。

- 创建/编辑角色：SOUL（人设）、系统提示词、模型绑定、会话模式偏好
- 按角色精细授权工具、技能、MCP，控制能力暴露面与 Token 成本
- 支持「项目团队」（project-team）形态，将多个 `project-*` 角色组合成 Plan 模式可调度的协作单元
- 配置变更自动热重载，不需重启服务

#### 消息渠道 `/channels`

打通飞书、Slack、Telegram、微信等 IM 平台，让 Agent 在用户惯用的协作工具中收发消息。

- 统一管理飞书、Slack、Telegram、微信（iLink）等已接入渠道的应用凭证、回调地址与启用状态
- 支持会话映射：将指定群聊 / 单聊与特定 Agent 绑定，未映射时由用户用 `/agent` 动态切换
- 飞书渠道支持流式卡片更新；其他渠道以文本与卡片形式同步对话与任务结果
- 内置回调地址校验、Token 刷新与连接诊断，便于排查推送失败问题

### 1.3 数据

#### 记忆文件 `/memory`

查看、编辑与导入导出 Agent 的长期记忆，是模型「记得用户」的事实底座。

- 按全局记忆与各 Agent 独立记忆分组，结构化展示 `workContext`、`personalContext`、`topOfMind`、`facts`、`history` 等字段
- 支持单条事实的查看、编辑、删除与置信度调整，避免脏数据污染上下文
- 提供 `memory.json` 导入导出，便于迁移、备份与跨实例同步
- 与「设置 → 记忆」轻量编辑互补，主页面适合批量梳理与全局视图

#### 定时任务 `/cron`

周期任务与一次性任务的调度中心，让 Agent 7×24 自动执行巡检、汇总、报告类工作。

- 支持每小时 / 每天 / 每周 / 一次性 / 自定义 Cron 五种调度模式
- 任务触发后会启动独立 Agent 会话执行 prompt，结果可推送到飞书等已配置渠道
- 提供「立即执行」「暂停 / 恢复」「执行历史」等运行时管理；历史以 JSONL 持久化便于审计
- 与托管任务互补：托管偏向多轮自主推进，定时偏向到点重复触发

### 1.4 扩展

#### Skills `/skills`

技能（`SKILL.md`）市场与本地管理：浏览、安装、启用、上传自定义技能。

- 以「已安装 / 搜索安装」分页呈现，支持全文检索与分类筛选
- 支持安装社区 `.skill` 归档、本地目录加载与 `skills/custom/` 自定义技能
- 启用 / 禁用单条技能控制其在 Agent 工具集中的可见性，配合「Agent 管理」做角色级白名单
- 内置 50+ 个公开技能，覆盖文档、研究、媒体、开发流程（含 `superpowers-*` 系列）等场景

#### 工具管理 `/tools`

内置工具与 MCP 服务器的统一治理面板。

- 查看内置工具清单、能力描述与启用状态（read / terminal / web_search / view_image 等）
- 添加 MCP 服务（stdio / SSE / HTTP），配置启动参数、OAuth 凭证与环境变量
- 支持 MCP 热重载、连接诊断、工具列表实时探活；服务异常会高亮告警
- 与「Agent 管理」联动：每个 Agent 可独立勾选可用工具与 MCP 子集

#### 任务中心 `/tasks`

复杂多 Agent 协作任务的总控台（受 `SHOW_TASK_CENTER_NAV` 开关控制可隐藏）。

- 创建大型任务后由 Supervisor 自动拆解为有向无环子任务图，分配给具备相应能力的 Agent
- 任务卡片实时展示状态（待开始 / 规划中 / 执行中 / 已暂停 / 已完成 / 失败 / 已取消）与进度
- 支持暂停、重试、改派、终止；可跳转到子任务的实时对话进行人工干预
- 与 Plan 模式互补：Plan 模式聚焦单会话内多步推进，任务中心聚焦跨会话、跨 Agent 的工程化协作

### 1.5 运维

#### 会话调试 `/observability`

EvoFlow 运行时的可观测性视图：调用链、Token 用量、慢调用与错误追踪。

- 按会话 / 任务 / Agent 维度查看模型调用、工具调用、子智能体派发的完整轨迹
- 提供 Top N Token 消耗、慢调用排行、错误聚类等诊断卡片
- 支持系统提示词、工具入参出参的 JSON 详情查看，便于复现与调优
- 与 `backend/docs/AGENT_TRACE_DEBUG_PLAYBOOK.md` 排障手册配套使用

### 1.6 底部

#### 面板设置 `/settings`

打开「设置弹窗」的入口，承载所有面板级偏好与轻量配置（详见 [二、设置弹窗](#二设置弹窗)）。

- 单击触发右侧覆盖式设置弹窗；URL 同步 hash（`#/settings?tab=xxx`），可直接外链到具体 Tab
- 弹窗内左侧导航 + 右侧多面板，按 Tab 懒加载，切换不重渲染避免高度闪动
- 所有设置自动持久化到面板偏好（`panel-settings`），跨实例隔离
- 与主导航部分入口（模型、消息渠道、记忆）功能互补：主菜单偏全屏管理，弹窗偏快速调整

## 二、设置弹窗

> 元数据来源：`evopanel/src/pages/settings/meta.js` 中的 `TAB_META`。各 Tab 由 `pages/settings/hub.js` 按需懒加载。

### 2.1 通用 `general`

应用级偏好的总控页，覆盖外观、行为、文件系统与数据目录等基础设置。

- 主题模式（日间 / 夜间 / 跟随系统）、字体大小、界面语言
- 工具审批策略：默认询问 / 自动批准 / 仅会话内授权；可对终端、删文件等高风险操作单独配置
- 记忆开关、虚拟沙箱路径（`/mnt/user-data/...`）、用户工作空间根目录
- 缓存清理、数据目录定位（`~/.evoflow/`），便于排障与备份

### 2.2 模型 `models`

设置弹窗内的快速模型管理面板，与主菜单 `/models` 共享同一数据但聚焦快速切换。

- 列出已配置的对话模型，支持快速切换默认主模型
- 显示连接状态与最近一次调用结果，便于排查鉴权或网络问题
- 创意媒体模型（生图、生视频、TTS）独立分组管理
- 与主菜单 `/models` 互通：主菜单偏精细配置，弹窗偏日常切换

### 2.3 环境变量 `env`

为 Agent 运行时（含 terminal 工具与技能脚本）注入持久化环境变量的中心。

- 以 `KEY=VALUE` 形式集中维护，避免在多个配置文件中分散管理
- 写入后对沙箱内的 terminal、技能脚本、子代理统一可见
- 支持敏感值掩码显示与单条删除；变更自动热生效，无需重启
- 适合放置 API Key、代理设置、第三方服务凭证等运行时依赖

### 2.4 IM 通信 `im`

IM 渠道的轻量配置入口，与主菜单 `/channels` 共享数据。

- 录入飞书 App ID / Secret、Slack Bot Token、Telegram Bot Token 等基础凭证
- 配置消息回调地址、事件订阅与机器人能力开关
- 支持「测试推送」一键校验渠道是否打通
- 复杂场景（会话映射、推送规则）建议在主菜单 `/channels` 全屏页面操作

### 2.5 记忆 `memory`

记忆文件的快速查看与编辑面板，与主菜单 `/memory` 共享数据。

- 按全局记忆 + 各 Agent 分 Tab 浏览，支持单条事实编辑与删除
- 适合在对话间隙快速修正错误事实或补充偏好
- 复杂操作（批量导入导出、跨 Agent 同步）建议在主菜单 `/memory` 进行
- 关闭弹窗后变更立即生效，下一轮对话即可注入更新后的记忆

### 2.6 快捷键 `shortcuts`

聊天与全局快捷键的自定义中心，预防与操作系统或其他应用冲突。

- 列出全部可绑定动作：发送消息、新建会话、打开命令面板、切换 Agent、停止生成等
- 支持单键、组合键、不同操作系统差异化绑定
- 冲突检测：与已知系统级快捷键重叠时给出告警提示
- 一键恢复默认；变更立即生效，无需重启面板

### 2.7 关于 `about`

应用版本、更新通道与法律声明的展示页。

- 显示当前 EvoPanel 版本、构建时间、Git Commit Hash 与平台信息
- 「检查更新」对接 GitHub Releases 与内置更新通道，支持一键下载新版
- 链接到官网、GitHub 仓库、文档站、Releases 与微信交流群
- 展示许可证信息（Evovex AI 非商业许可证 1.0）与上游致谢（NOTICE）

## 三、隐藏路由入口

> 来源：`evopanel/src/router.js` 的动态段路由。这些路由不出现在左侧导航，但可由列表跳转、URL 直达或外链唤起。

### 3.1 任务详情 `/task/:id`

任务中心单个任务的执行视图，是主菜单 `/tasks` 的下钻页。

- 展示任务计划全文：Supervisor 生成的子任务拆解、依赖关系与目标产出
- 子任务卡片支持单独暂停 / 重试 / 改派 / 查看结果
- 右侧实时调度日志：模型调用、工具调用、子代理派发的事件流
- 「实时对话」按钮跳转到该任务关联的聊天线程，便于人工干预

### 3.2 项目详情 `/project/:id`

工作项目的详情页，承载该项目下的会话、记忆与任务隔离视图。

- 展示项目元信息：名称、根目录、绑定 Agent、记忆作用域
- 列出该项目下的所有会话与任务，便于按项目维度回顾历史
- 支持项目级配置覆盖：默认 Agent、默认模型、工具白名单
- 与全局工作空间互补：项目作用域更小、隔离更强，适合多客户 / 多产品场景

### 3.3 工作流详情 `/workflow/:id`

工作流（多步骤编排模板）的设计与执行视图。

- 可视化展示工作流步骤、依赖关系与触发条件
- 支持步骤级参数配置、Agent 绑定与产物路径定义
- 提供试运行入口，便于在投产前验证编排正确性
- 历史运行记录可追溯，每次执行的输入输出均可回看

### 3.4 Agent 团队详情 `/agents/team/:code`

项目团队（`project-*` 系列预设角色组合）的详情页，是 Plan 模式协作的能力清单。

- 展示团队成员（Agent 列表）、各自人设（SOUL）、绑定模型与可用工具白名单
- 标注成员间的协作分工：哪些角色负责规划、哪些负责执行、哪些负责验收
- 支持单个成员的快速跳转与配置编辑，避免在 `/agents` 全列表中反复检索
- 与 Plan 模式联动：主控 `plan` 工具调度子任务时，依据本页定义的能力画像分发

### 3.5 React 版聊天 `/chat-react`

> 实验性路由，默认不暴露给终端用户，仅用于内部灰度。

与 `/chat` 并列的下一代聊天容器，承载 React 重构后的消息流与 Composer。

- 与 `/chat` 共享会话数据、Agent、模型与场景配置，可无缝切换
- 采用虚拟滚动与增量渲染，长会话的滚动与重渲染性能更优
- 工具调用块、子代理执行块、Plan 阶段卡片等富交互组件原生支持
- 默认入口仍为 `/chat`；本路由用于灰度验证与 React 版独立调试
