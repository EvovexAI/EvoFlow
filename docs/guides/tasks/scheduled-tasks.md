# 自动化操作指南

自动化是 EvoFlow 的**到点重复触发**能力——你定一个时间规则（每天 9 点、每周一、每小时第 30 分钟、cron 表达式……），到点系统就自动起一个 Agent 会话执行你写好的指令，结果可推送到飞书等渠道。适合日报、巡检、监控、定期数据汇总等"机械重复"工作。

---

## 一、何时用自动化

自动化、托管智能体、任务中心三者最容易混，先用一张表区分：

| 你想做的事 | 推荐路径 | 为什么 |
|----------|---------|--------|
| **到点重复**触发一段固定逻辑（每天 9 点出日报） | **自动化** | cron / RRULE 调度，机器人按时干 |
| 一个开放目标，**连续多轮自驱**直到完成（持续监控、长任务推进） | **托管智能体** | 7×24 自驱，到边界自动停 |
| 复杂多角色协作的**一次性大工程**（重构、调研报告） | **任务中心** | 子任务图、并行派发、DAG 监控 |
| 多步骤可交互的任务（写代码、改文档） | **Plan 模式** | 在聊天里直接拆步、确认、执行 |

自动化的核心特征：**"频率固定、内容相对稳定、不需要人盯着"**——例如"每天早上 8 点汇总昨日 PR、生成日报推送到飞书"、"每周一巡检线上日志"、"每月 1 号生成账单"。

---

## 二、入口与首页

点击左侧菜单「数据」分组下的「自动化」进入调度管理页面。

**页面布局**：

- **顶部状态栏**：调度器运行状态、活跃任务数、暂停任务数、总任务数
- **任务卡片列表**：每张卡片展示
  - 图标、任务名称、执行频率（如"每天 09:00"）、任务描述（前两行）
  - 运行状态徽标（运行中 / 已暂停 / 一次性已完成）
  - 飞书推送标识（已开启则显示对应 chat_id）
- **快捷操作**：
  - ▶ 运行：立即手动触发一次
  - ⏸ 暂停 / ↻ 恢复：控制运行状态
  - 📋 历史：查看所有执行记录
  - ✎ 编辑：修改任务配置
  - ✕ 删除：移除任务

每个卡片都对应磁盘上一份 TOML 文件（详见 [七、与 TOML 文件的关系](#七与-toml-文件的关系)），GUI 操作 = 改 TOML。

## 三、创建自动化

### 方式一：界面按钮（推荐）

点击右上角「新建任务」，按表单填写：

#### 1. 基本信息

| 字段 | 必填 | 说明 |
|------|------|------|
| **任务名称** | 是 | 自定义标识，便于在列表里识别 |
| **任务描述（Prompt）** | 是 | 每次执行时发给 Agent 的指令，**越具体越好** |

> Prompt 写法建议：直接写"你要做什么、产出什么"，例如"汇总昨日 GitHub 上 EvoFlow 仓库的 PR 与 Issue 变化，生成 Markdown 日报"。**不要**写"请你"、"麻烦"、"如果可以的话"等寒暄词，AI 会照单全收浪费 Token。

#### 2. 调度配置（5 种模式）

| 模式 | 说明 | 例子 |
|------|------|------|
| **每小时** | 选每小时第几分钟触发 | 每小时第 30 分钟 |
| **每天** | 选具体时间 | 每天 09:00 |
| **每周** | 选星期几 + 时间 | 周一至周五 18:00 |
| **一次性** | 选具体日期时间，支持快捷选项（现在 / +1h / 明早 9:00 / 今晚 22:00） | `2026-07-01 10:00` |
| **自定义 Cron** | 5 字段 cron 表达式 `分 时 日 月 星期` | `0 9 * * 1-5`（周一至周五 9:00） |

> 表单选完任意模式后，下方会显示"下次将在 XX 触发"的预览，确认无误再保存。

#### 3. 高级设置

| 选项 | 默认 | 说明 |
|------|------|------|
| **工作目录** | 空 | 指定 Agent 执行时的工作目录；不填则用 Gateway 默认目录 |
| **任务超时时间** | 60 分钟 | 单次执行最长时长，范围 1–1440 分钟 |
| **飞书推送** | 关 | 开启后每次执行结果自动推送到指定飞书 chat_id |
| **记忆** | **关** | 开启后任务保留上下文记忆，多次执行可复用 |

> ⚠️ **关于记忆开关**：默认是**关闭**的。多数自动化（如日报）每次独立执行更干净，不需要记忆。仅当任务需要"接着上次结果继续"时再开启——例如"持续追踪某项目进度，每周补充新增内容"。

填完点「创建任务」即可。任务创建后立即进入调度器，到点自动触发。

### 方式二：对话直接触发

不想离开聊天页时，直接在聊天里描述需求：

```
帮我创建一个自动化，每天早上 9 点生成昨天的系统运行统计报告，
推送到飞书运维群（chat_id: oc_xxxxx）。
```

```
每周五下午 6 点自动执行项目代码安全扫描，结果存到 outputs/。
```

```
创建一个一次性任务，明天上午 10 点执行数据库备份脚本。
```

主智能体会自动识别周期、内容、推送需求，**返回一份方案确认条**——你点"确认"才正式创建并启用。

## 四、对话式管理已有任务

除了创建，**查询、暂停、恢复、删除、查历史**也都能在聊天里说人话完成：

| 你说 | 实际行为 |
|------|---------|
| `查看所有正在运行的自动化` | 列出 `status=active` 的任务 |
| `查询最近执行失败的自动化` | 过滤近期失败的执行记录 |
| `暂停 ID 为 xxx 的自动化` | 设 `status=paused` |
| `恢复 xxx 自动化` | 设回 `status=active` |
| `删除 xxx 自动化` | 移除 TOML 文件 |
| `查看 xxx 自动化最近的执行记录` | 读取 `*_history.jsonl` |

适合日常运维——不用切到 GUI 就能批量管。

## 五、执行历史

点任务卡片的「历史」按钮，看完整执行记录：

| 字段 | 说明 |
|------|------|
| **执行时间** | started_at（UTC） |
| **状态** | 成功 / 失败 / 超时 |
| **触发方式** | `schedule`（自动到点）/ `manual`（手动触发） |
| **执行耗时** | duration_seconds |
| **输出内容** | Agent 返回的最终结果（截断显示，完整内容点详情） |
| **错误信息** | 失败时的栈信息 |

历史以 JSONL 格式持久化在 `~/.evoflow/tasks/automations/{task_id}_history.jsonl`，可用 `tail -f` 实时跟、可 `jq` 二次分析。

## 六、飞书推送

自动化最常见的搭配——任务到点跑完后把结果推到飞书群。

### 开启推送

1. 创建 / 编辑任务时，在「高级设置」勾选 **飞书推送**
2. 填写 **chat_id**（格式 `oc_xxxxxxxxxxxx`）
3. 保存后下次执行就会推送

### 推送内容

每次执行结束后，飞书会收到一条卡片消息：

- 任务名称、执行状态（✅ 成功 / ❌ 失败 / ⏱ 超时）
- 执行耗时
- Agent 最终输出（Markdown 渲染）
- 失败时附错误详情，便于排查

### 获取飞书 chat_id

| 方式 | 操作 |
|------|------|
| **群设置** | 飞书群右上角「设置」→ 群信息，复制以 `oc_` 开头的群 ID |
| **开放平台 API** | `GET /open-apis/im/v1/chats`，需企业自建应用权限 |
| **让机器人自报** | 把已配置的 EvoFlow 飞书机器人拉进群，在群里 @ 机器人发"群 ID"，它会回 chat_id |

详细配置见 [飞书集成](../integration/feishu-integration.md)。

### 推送失败排查

| 现象 | 检查 |
|------|------|
| 完全没收到 | 飞书渠道是否启用、机器人是否已加入目标群 |
| 报 "chat_id not found" | chat_id 拼写、机器人权限范围 |
| 报 "rate limit" | 飞书 API 限流，调大任务执行间隔 |
| 收到但内容为空 | Agent 执行成功但无输出，检查 prompt 是否要求了产出 |

## 七、与 TOML 文件的关系

GUI 上的每个自动化都对应磁盘上一份 TOML 文件：

```
~/.evoflow/tasks/automations/
├── daily-report.toml           # 每日日报
├── weekly-scan.toml            # 每周巡检
└── daily-report_history.jsonl  # 执行历史
```

**GUI 与 TOML 是同步的**：

- 在 GUI 创建 / 编辑 / 删除 → 自动写 TOML
- 直接编辑 TOML 文件 → 调度器下次 tick 自动加载（默认 30 秒间隔）

这意味着你可以：

- **跨实例同步**：把 `~/.evoflow/tasks/automations/` 整目录拷到另一台机器
- **批量管理**：用脚本一次性生成几十个任务的 TOML
- **版本控制**：把目录加进 git，团队共享任务定义

TOML 字段全集、字段语义、环境变量、API 详见 [自动化任务调度](./automation-scheduler.md)。

## 八、常见问题

**Q：任务到点了但没执行？**
依次检查：

1. 任务状态是不是 `运行中`（不是 `已暂停`）
2. 调度器是否运行——顶部状态栏看，或 `curl http://localhost:8001/api/automation/scheduler/status`
3. 是否被并发数限制（默认 `EVOFLOW_AUTOMATION_LANGGRAPH_MAX_CONCURRENT=2`）
4. IM 渠道服务是否运行（调度器依赖它）

**Q：cron 表达式怎么写？**
EvoFlow 用标准 5 字段 cron：`分 时 日 月 星期`。

- `0 9 * * *` 每天 9:00
- `*/30 * * * *` 每 30 分钟
- `0 9 * * 1-5` 工作日 9:00
- `0 0 1 * *` 每月 1 号 0:00

不会写？直接在表单里用"每天 / 每周 / 每小时"模式，系统会帮你生成对应 cron。

**Q：一次性任务执行完了，能改成重复任务吗？**
不能直接改。一次性任务执行后 `once_fired=true` 且自动 `paused`，需要新建一个重复任务。

**Q：怎么知道任务下一次什么时候触发？**
新建/编辑表单底部的"下次触发预览"会显示；任务列表卡片上也会标注"下次执行：YYYY-MM-DD HH:MM"。

**Q：临时停用所有自动化？**
设环境变量 `EVOFLOW_AUTOMATION_SCHEDULER=0` 并重启 Gateway，整个调度器停摆但 TOML 不删。

**Q：任务能引用上一次的执行结果吗？**
开启高级设置里的"记忆"开关后可以——Agent 会按 sticky 模式复用同一个 LangGraph 线程，自动看到上次产出。注意默认是关的。

**Q：任务跑太久卡住了？**
单次执行最长 `任务超时时间`（默认 60 分钟，最长 1440），超时自动终止并记 `failed`。如果常超时，把 prompt 拆细或抬高超时上限。
