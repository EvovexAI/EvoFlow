# 定时任务与通知推送指南
---
## 页面入口
点击左侧菜单栏「扩展」分类下的「定时任务」进入自动化调度管理页面。

---
## 定时任务管理
### 页面概览
- 顶部状态栏：显示调度器运行状态、活跃任务数、暂停任务数、总任务数统计
- 任务列表：每个任务卡片展示图标、任务名称、执行频率、任务描述、运行状态（运行中/已暂停/一次性）
  - 已开启飞书推送的任务会显示飞书推送标识和对应的chat_id
- 任务操作：
  - ▶ 运行：手动触发任务执行
  - ⏸ 暂停 / ↻ 恢复：控制任务的运行状态
  - 📋 历史：查看任务的所有执行记录
  - ✎ 编辑：修改任务配置
  - ✕ 删除：删除不需要的任务

---
## 创建定时任务
点击页面右上角「新建任务」按钮，按照以下步骤配置：

### 1. 基本信息
- **任务名称**：自定义任务标识，便于识别
- **任务描述（Prompt）**：详细描述每次执行时需要完成的任务内容

### 2. 调度配置
支持5种调度模式：
| 模式 | 配置说明 |
|------|----------|
| **每小时** | 选择每小时的第几分钟触发，例：每小时第30分触发 |
| **每天** | 选择具体的执行时间，例：每天 09:00 执行 |
| **每周** | 选择星期几和具体时间，例：每周一至周五 18:00 执行 |
| **一次性** | 选择具体的日期时间执行，支持快捷选择：现在、+1小时、明早9:00、今晚22:00 |
| **自定义 Cron** | 输入Cron表达式精确控制触发时间，格式：`分 时 日 月 星期`，例：`0 9 * * 1-5`（周一至周五 9:00） |

### 3. 高级设置
- **工作目录**（可选）：指定任务执行的工作目录
- **任务超时时间**：设置任务最长执行时间，范围1-1440分钟
- **飞书推送**：开启后任务执行结果会自动推送到指定的飞书chat_id
- **记忆**：开启后任务会保留上下文记忆，多次执行时可以复用之前的信息

配置完成后点击「创建任务」即可启用。

## 对话式管理定时任务
不需要进入定时任务页面，直接在聊天对话中就可以完成所有定时任务的创建和管理操作，智能体会自动引导补全必要信息：

### 创建定时任务
直接描述需求即可，例如：
```
帮我创建一个定时任务，每天早上9点生成昨天的系统运行统计报告，结果推送到飞书群
```
```
每周五下午6点自动执行项目代码安全扫描
```
```
创建一个一次性任务，明天上午10点执行数据库备份
```

智能体会自动识别任务的执行周期、内容、推送需求，确认后自动创建并启用。

### 管理定时任务
同样可以通过对话管理已有任务：
- **查询**：`查看所有正在运行的定时任务` / `查询最近执行失败的定时任务`
- **操作**：`暂停ID为xxx的定时任务` / `恢复xxx定时任务` / `删除xxx定时任务`
- **查看历史**：`查看xxx定时任务最近的执行记录`

---
## 执行历史查看
点击任务卡片的「历史」按钮，可以查看所有执行记录：
- 每次执行的时间、状态（成功/失败/超时）
- 执行输出内容和错误信息
- 触发方式（自动/手动）
- 执行耗时

---
## 飞书推送配置
### 开启推送
在定时任务的高级设置中：
1. 勾选「飞书推送」启用开关
2. 填写要推送的飞书chat_id（格式通常为`oc_xxxxxx`）
3. 保存配置后，任务每次执行完成后会自动推送结果到对应飞书会话

### 说明
- 推送内容包含任务执行状态、执行结果、耗时等信息
- 如果执行失败会同时推送错误详情，便于排查问题
