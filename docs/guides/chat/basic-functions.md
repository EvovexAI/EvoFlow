# 基础功能使用指南
---
## 1. 对话功能
- **多模态输入**：支持发送文本、图片、文件，支持视觉的模型会自动识别图片内容
- **流式输出**：回复实时逐字渲染，不需要等待全部生成完成即可阅读
- **工具调用可视化**：AI调用工具、执行代码的过程会实时展示，可展开查看完整运行日志和返回结果
- **会话管理**：左侧侧边栏可查看所有历史会话，支持搜索、删除会话，点击即可切换上下文
- **线程模式**：支持在同一个会话中创建多个独立线程，适合同时处理多个相关但隔离的子任务

---
## 2. 文件上传与解析
点击聊天框右侧上传按钮，或直接拖拽文件到聊天窗口即可上传：
- **文档类**：支持PDF/Word/Excel/PPT/markdown/txt等格式，自动解析文本内容
- **代码类**：支持所有常见编程语言的代码文件、压缩包，甚至整个项目文件夹，AI可以直接读取分析代码结构和内容
- **图片类**：支持JPG/PNG/GIF等常见图片格式，视觉模型可以理解图片内容，支持截图直接粘贴上传
- 上传后AI会自动识别文件内容，可直接针对文件内容提问、要求分析、修改、重构等操作

## 3. Plan模式使用

针对复杂多步任务，推荐在顶栏使用 **「任务规划」** 场景（会自动开启任务协作 / Plan 协作）。

### 与 Cursor Plan 的差异（简要）

| 维度 | Cursor | EvoFlow |
|------|--------|---------|
| 计划存放 | 工作区 Markdown 文件 | **任务表**（`plan_goal` / `plan_steps` 等，由 `plan` 工具落库） |
| 审阅计划 | 虚拟/可保存的 `.md` | 聊天区 plan 工具摘要 + **「查看计划」** 弹窗 + 协作侧栏子任务 |
| 开始执行 | Build 按钮 | 输入框上方 **「开始执行」**（授权后由 Supervisor 按 DAG 派发） |

EvoFlow **不会**把工作区 `plan.md` 当作权威数据源；改计划请通过「修改计划」让助手重新调用 `plan` 更新任务表。

### 使用流程

1. 切换到 **任务规划** 场景，发送需求；助手会先摸底（`subagent`）再调用 **`plan`** 落库。
2. 在 **`plan` 工具块** 查看目标与步骤数摘要；完整正文点 **「查看计划」**；子任务在 **协作侧栏** 同步。
3. 在输入框上方点击 **「开始执行」** 授权；后端写入 `execution_authorized` 并派发 `supervisor start_execution`（有子任务时由网关直接派发，无需再在对话里重复说「开始执行」）。
4. 需改计划：点 **「修改计划」** 或说明意见，助手 `plan(...)` 更新任务表并同步子任务结构。
5. **刷新页面** 后，若计划已定稿且未执行，确认条应从历史消息与任务快照恢复（依赖 `taskId` 与 plan 工具落库记录）。

### 常见问题

- **只看到「计划已就绪」文字、没有按钮？** 说明前端未挂上确认条；请强刷或重新 build 前端，并确认协作侧栏是否已有子任务。
- **计划正文在哪？** 不在单独 Markdown 文件，而在任务表；用「查看计划」或任务详情中的计划视图。

## 4. 记忆管理
点击左侧菜单栏「记忆」进入记忆管理页面：
- **全局记忆开关**：可以开启/关闭全局记忆功能，开启后AI会自动提取对话中的有用信息作为长期记忆，跨会话复用
- **记忆查看**：可以浏览所有记忆条目，分为工作上下文、个人偏好、知识库、长期目标等分类
- **记忆管理**：支持手动添加、编辑、删除记忆条目，也可以一键清除全部记忆
- **会话级记忆**：每个会话也有独立的上下文记忆，不会混淆到全局记忆中

---
## 5. 多项目隔离
点击左侧菜单栏「项目」进入项目管理页面：
- 可以创建多个独立工作项目，不同项目的会话、记忆、配置、自动化、托管任务完全隔离
- 每个项目可以配置独立的默认智能体、工具权限、MCP服务器、工作目录
- 聊天页面顶部可以快速切换项目，适合不同工作场景、不同客户项目分开使用，互不干扰
- 支持项目导出、导入功能，方便备份和迁移

---
## 6. 个性化设置
点击左下角「设置」按钮进入设置页面：
- **外观设置**：支持亮色/暗色/自动跟随系统三种主题切换，支持调整字体大小、行高、代码主题
- **语言设置**：支持中英文界面切换，AI回复语言会自动跟随界面语言，也可以手动指定
- **模型设置**：可以设置默认使用的模型，配置模型的温度、最大生成长度、推理努力程度等参数
- **快捷键设置**：可以自定义常用操作的快捷键，支持全局快捷键唤起窗口、新建会话、发送消息等
- **通知设置**：可以配置任务完成、异常告警、托管任务进度等通知的触发方式和渠道
