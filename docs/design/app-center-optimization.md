| # | 问题 | 优先级 | 影响面 |

## 1. P0：主路径 — 卡片直接显示「运行」按钮

### 现状

- 卡片点击 → 进入画布编辑
- 「填参运行」藏在 ··· 菜单里（仅已发布应用才有）
- 用户误以为「画布」是日常入口

### 优化

- 已发布应用的卡片底部直接显示「▶ 运行」按钮
- 点击卡片本体仍进画布（编辑流程）
- 草稿/已归档卡片保持现有行为（进画布）

### 验收

- [ ] 已发布卡片底部有「▶ 运行」按钮，点击跳 `/apps/{id}/run`
- [ ] 点击卡片其他区域仍进画布
- [ ] 草稿卡片无「运行」按钮

---

## 2. P0：价值感知 — 适用场景 + 推荐模板

### 现状

- 页面描述："把多步智能体流程做成可复用应用：编排 → 画布调试 → 发布后填参运行。"
- 这是操作说明，不是价值说明
- 新用户不知道什么时候该用应用

### 优化

- 页面顶部增加「适用场景」提示条
- 空状态增加「从模板创建」入口（竞品分析、代码审查、周报生成）

### 验收

- [ ] 页面顶部显示适用场景提示
- [ ] 空状态有模板快捷创建按钮

---

## 3. P1：空状态引导优化

### 现状

```
点下方创建 → 在画布选智能体、写步骤 → 点「调试」试跑 → 满意再发布。
```

对新用户来说步骤太多，认知负担重。

### 优化

- 简化文案，突出最快路径
- 增加「从模板创建」和「从对话生成」两个快捷入口

### 验收

- [ ] 空状态文案 ≤ 2 行
- [ ] 有模板创建按钮
- [ ] 有从对话生成按钮（如果后端支持）

---

## 4. P1：卡片信息密度优化

### 现状

每张卡片显示：图标、名称、状态徽章、步骤数、参数数、执行模式、描述、···菜单

### 优化

- 移除「执行模式」元信息（对运行者无意义）
- 步骤数和参数数合并为一行（如 "3 步 · 2 参数"）
- 描述限高 2 行，超出省略

### 验收

- [ ] 卡片元信息 ≤ 2 项
- [ ] 描述超 2 行省略

---

## 5. P1：操作路径优化

### 现状

- 「运行」在 ··· 菜单里
- 「编辑信息」在 ··· 菜单里
- 「归档」在 ··· 菜单里

### 优化

- 「运行」提升为卡片底部主按钮（P0 已做）
- ··· 菜单保留：编辑信息、归档/恢复、删除
- 已发布卡片增加「编辑」快捷入口（直接进画布）

### 验收

- [ ] 运行按钮在卡片底部
- [ ] ··· 菜单项 ≤ 4 个

---

## 6. P2：从对话/任务沉淀入口

### 现状

- 后端支持 `POST /api/apps` 的 `source: "from_task"` 方式
- 前端没有入口

### 优化

- 应用中心「新建」下拉菜单增加「从任务沉淀」选项
- 弹窗选择已有任务 → 自动提取参数 → 创建应用

### 验收

- [ ] 新建按钮有下拉菜单
- [ ] 可选择任务并创建应用

---

## 7. P2：筛选 Tab 文案优化

### 现状

- 「进行中」— 含义模糊（是运行中？还是未归档？）
- 「已归档」— 清晰
- 「全部」— 清晰

### 优化

- 「进行中」→「活跃」或直接去掉，默认只显示活跃应用
- 考虑增加搜索框（按名称/描述搜索）

### 验收

- [ ] Tab 文案更直观
- [ ] 可选：增加搜索框

4. **P1-2**：卡片信息密度优化 → `apps.js` 卡片 meta

7. **P2-2**：筛选 Tab 文案 + 搜索框 → `apps.js` 筛选区
