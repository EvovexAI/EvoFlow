# 创建定时自动化任务

> **比如你可以让 EvoFlow 定时做这些事**：
>
> - **每周五下午 5 点**：自动汇总本周项目进展，生成周报推给你
> - **每天上午 9 点**：检查服务器健康状态，异常就告警
> - **每月 1 号**：生成上个月的数据分析报告
>
> 在「自动化」面板里点几下，写好指令和 cron 表达式，到点自动跑。结果可以推飞书、Slack 等渠道。
>
> 自动化 vs 智能体员工：自动化是"到点跑固定 Prompt"，员工是"有职责、有审批、有交班的岗位"。

## 你将学到什么

- 理解自动化任务（Automation）是什么：通过 cron / rrule 规则定时触发的 Agent 任务
- 通过**面板界面**创建和管理自动化任务
- 在 GUI 中配置推送结果
- 手动触发测试和查看执行历史

## 前置条件

- 已完成 [安装](../getting-started/installation.md)
- EvoFlow 服务正在运行
- （可选）已接入消息渠道（飞书、Slack 等）用于推送结果

## 预计用时

10 分钟

## 步骤

### 1. 理解自动化任务

自动化任务（Automation）允许你定义由调度器定期执行的 Agent 任务。每次触发时，调度器会启动一个 Agent 会话，执行你定义的 prompt，并将结果推送给你指定的目标（如飞书群聊）。

调度器支持三种时间规则：

| 规则类型 | 格式示例 | 说明 |
|---------|---------|------|
| **每小时** | 每小时第 30 分触发 | 周期短、频率高时用 |
| **每天/每周** | 每天 9:00 / 每周一 8:00 | 日报、周报等固定节律 |
| **自定义 Cron** | `0 9 * * 1-5` | 灵活控制，工作日才跑 |

### 2. 创建第一个自动化任务（推荐：GUI 面板）

这是最常用的方式，你不需要编辑任何配置文件。

#### 2.1 打开自动化面板

在 EvoFlow 左侧栏点击 **「自动化」**（路由 `#/cron`）。

#### 2.2 点击新建任务

点 **＋ 新建任务**，弹窗有 3 个 Tab。

#### 2.3 填写基本信息

在 **Tab 1 · 基本信息** 中：

| 字段 | 必填 | 说明 |
|------|------|------|
| **任务名称** | 是 | 如：每日健康检查 |
| **任务描述 (Prompt)** | 是 | 每次发给 Agent 的指令，写清做什么、产出什么 |

Prompt 宜具体，少寒暄。例如：

```
请执行以下巡检任务：
1. 检查最近 7 天的应用日志，找出所有 ERROR 级别日志
2. 分析 API 响应时间，标记平均响应时间 > 500ms 的接口
3. 检查磁盘使用率超过 80% 的目录
4. 生成巡检报告，列出异常项和建议操作
```

#### 2.4 配置调度时间

在 **Tab 2 · 调度配置** 中，选择频率：

| 模式 | 说明 |
|------|------|
| **每小时** | 每小时第 MM 分触发 |
| **每天** | 选时刻（含上下午） |
| **每周** | 星期 chips + 时刻 |
| **一次性** | 选日期时间；快捷：**现在** / **+1小时** / **明早 9:00** / **今晚 22:00** |
| **自定义 Cron** | 五字段：`分 时 日 月 星期` |

底部有调度摘要预览（如「每天 09:00 执行」）。

#### 2.5 配置高级选项（可选）

在 **Tab 3 · 高级设置** 中：

| 选项 | 默认 | 说明 |
|------|------|------|
| **工作目录** | 空 | 可「选择文件夹」；空则用 Gateway 默认 |
| **任务超时时间（分钟）** | **30** | 单次上限，常见范围 1–1440 |
| **完成后推送结果** | 关 | 打开后选 **推送渠道**（需先配好消息渠道） |
| **记忆 · 启用** | 关 | 多数日报类任务保持记忆关闭更干净 |

#### 2.6 保存

点 **创建任务**，自动化任务就建好了。

### 3. 测试运行

在自动化任务卡片上点击 **▶ 运行**，手动触发一次：

1. 任务状态变为「执行中」
2. Agent 按 prompt 执行任务
3. 执行完成后，结果会推送到已配置的渠道
4. 点 **📋 历史** 查看执行记录

### 4. 管理自动化任务

| 操作 | 说明 |
|------|------|
| **▶ 运行** | 立即手动触发一次（不改调度） |
| **⏸ 暂停 / ↻ 恢复** | 控制是否参与到点调度 |
| **📋 历史** | 查看执行记录 |
| **✎ 编辑** | 修改配置 |
| **✕ 删除** | 移除任务（需确认） |

### 5. 对话创建（快捷方式）

不想打开面板？在聊天里直接说：

> 帮我建一个每天 9 点检查服务器健康的自动化

AI 会帮你确认配置并创建，和 GUI 创建效果一样。

## 备选方案：通过 TOML 配置文件创建

如果你**习惯编辑配置文件**，或者需要批量管理多个自动化任务，也可以使用 TOML 方式。

> 以下内容为**替代方案**，对大多数用户来说直接用 GUI 面板更简单。需要了解 TOML 完整字段和运维配置的，参考 [自动化任务调度指南](../guides/tasks/automation-scheduler.md)。

### 创建任务目录

```bash
mkdir -p ~/.evoflow/tasks/automations/
```

### 编写 TOML 文件

创建一个文件 `~/.evoflow/tasks/automations/daily-report.toml`：

```toml
name = "每日项目进度日报"
description = "每天早上 9 点自动汇总昨日工作进展，并推送到飞书群组"
prompt = """
请生成一份今日的工作日报，包含以下内容：
1. 昨日完成的开发任务
2. 当前进行中的工作
3. 需要关注的问题或风险
4. 今日计划
"""
schedule = "0 9 * * *"
workspace = "/workspace/my-project"
memory_enabled = false
```

### 常用字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 任务名称 |
| `prompt` | 是 | 发送给 Agent 的指令 |
| `schedule` | 是 | cron 表达式（如 `0 9 * * *`） |
| `workspace` | 否 | Agent 执行时的工作目录 |
| `memory_enabled` | 否 | 是否使用记忆，默认 `false` |

完整字段参考见 [自动化任务调度指南](../guides/tasks/automation-scheduler.md)。

## 你完成了！

你已成功创建了一个定时自动化任务。EvoFlow 调度器会在到达设定时间时自动执行该任务，并将结果推送到配置的渠道。
