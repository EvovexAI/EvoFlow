# 案例八：Supervisor 模式（超级总控智能体）

> **比如你要做一个"全栈项目开发"，需要不同角色分工**：
>
> 1. 让 `project-architect` 出架构方案
> 2. 让 `project-implementer` 写代码
> 3. 让 `project-tester` 跑测试
> 4. 让 `project-reviewer` 审查代码
>
> Supervisor 模式自动拆解任务、分配不同 AI 角色去干、监督进度、出错了自动调整方案。
>
> Supervisor 模式需要先创建好自定义智能体（案例二）作为子智能体候选，通常配合 Plan 模式使用。

## 场景说明

当你面对一个复杂任务，需要多个专业角色协同完成时，Supervisor（超级总控智能体）模式就派上用场了。超级总控智能体会自动拆解任务、分配给多个子智能体、监督执行进度，并在出错时自动调整方案。

## 前置准备

- 已完成[安装](../getting-started/installation.md)
- 已配置至少一个 AI 模型
- 已创建至少一个自定义智能体（作为子智能体候选）
