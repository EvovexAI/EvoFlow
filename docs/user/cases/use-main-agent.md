# 案例一：雇佣运维员工——服务器巡检值班

> **场景：你维护 3 台服务器（gateway、langgraph、postgres），每天都要检查是否健康，有异常要告警。**
>
> 以前：每天 SSH 登录、敲命令、人工看日志……重复劳动。
>
> 用智能体员工：雇一个"运维助手"员工，定时巡检、异常告警、下班交班报告，你只用在待审批 Tab 拍板。

## 操作步骤

### 1. 创建运维角色

在「智能体」页创建角色，人设示例：

```
你是一个专业的运维工程师，擅长检查服务器健康状态、分析日志、排查故障。
检查时先确认服务可达性，再检查关键指标，最后汇总报告。
异常时先分析根因再建议修复方案，不轻易下结论。
```

配好 `terminal` 工具（用于 curl / ping 检查）和飞书推送权限。

### 2. 雇佣为员工

进入「智能体员工」页，点「雇佣员工」：

| 字段 | 填写示例 |
|------|----------|
| **绑定智能体** | 选上一步创建的运维角色 |
| **岗位名称** | 服务器巡检 |
| **职责** | 每 30 分钟检查 gateway / langgraph / postgres 健康状态；异常时分析根因并推送飞书告警；下班前写巡检报告 |
| **管辖工作空间** | 选一个目录，用于存放巡检报告 |
| **自主权** | **谨慎型**（所有事项先问你，推荐上手用） |
| **上岗节奏** | 每 30 分钟 |
| **上班时间** | 09:00–20:00 |

### 3. 开值班

值班台点「开启值班」，员工开始上岗。

### 4. 日常处理

- **09:00**：员工自动上岗，开始巡检
- **09:30**：第一轮完成，巡检报告写到工作空间 `docs/roles/<code>/<日期>/` 下
- **遇到异常**：员工暂停，待审批 Tab 出现事项——你点「同意」才执行告警
- **20:00**：写交班报告，下班

你每天只需要打开「待审批」Tab，看看有没有事项，同意或驳回即可。

## 核心要点

| 概念 | 说明 |
|------|------|
| **岗位 ≠ 角色** | 角色是"会什么"，岗位是"什么时候主动干、干到哪" |
| **自主权三档** | 谨慎型（全审批）/ 平衡型（中风险以上审批）/ 全自动（仅极高风险审批） |
| **交班报告** | 写在工作空间 `docs/roles/<agent_code>/<YYYYMMDD-HH>/` 下 |
| **立即开工** | 不受上班时间限制，随时手动触发一轮值班 |
