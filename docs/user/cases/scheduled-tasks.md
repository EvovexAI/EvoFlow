# 案例五：全栈项目交付——Plan + 任务中心看板

> **场景：你要做一个全栈 Todo 应用，从方案设计到编码到测试到部署，涉及多个环节。**
>
> 用 Plan 模式发起，用项目团队（6 个专业角色）分工协作，在任务中心看板跟踪进度，跑通后存为应用下次复用。

## 操作步骤

### 1. 用 Plan + 项目团队发起

聊天底部选 **Plan** 模式，描述任务：

```
帮我做一个全栈 Todo 应用，使用 FastAPI + React + SQLite。
使用项目团队协作完成：先让 architect 出方案，
implementer 写代码，tester 写测试，reviewer 审查。
```

AI 收到后，会调用 `plan` 工具生成多步骤任务，每个步骤自动绑定对应的 `project-*` 角色。

### 2. 确认计划

查看 AI 出的计划，确认步骤无误后点「**开始执行**」。

### 3. 在任务中心看板跟踪进度

切到「任务中心」，来源 Tab 选「对话」，可以看到：

| 列 | 内容 |
|----|------|
| **执行中** | 方案设计正在由 architect 执行 |
| **待办** | 开发、测试、审查等步骤排队中 |
| **已完成** | 已完成的步骤 |
| **失败** | 如果某步骤出错，自动重试 3 次后标记失败 |

### 4. 查看工作流 DAG

点击任务详情 →「查看工作流」，可以看到完整的 DAG 图：

```
[方案设计] → [代码开发] → [单元测试] → [代码审查] → [部署]
   (architect)    (implementer)  (tester)      (reviewer)    (implementer)
```

每个节点可点击查看该步骤的对话和产物。

### 5. 暂停 / 纠偏

如果开发步骤卡住了：

1. 在任务中心点「暂停」
2. 回到对话补充指令（如"数据库改用 PostgreSQL"）
3. 点「继续」——上下文和中间产物保留，继续执行

### 6. 交付后存为应用

全流程跑通后，在任务详情点「**另存为应用**」：

- 参数化：`项目名称`、`数据库类型`、`前端框架`
- 以后填参就能生成一个完整的项目脚手架

## 任务中心核心操作

| 操作 | 什么时候用 |
|------|-----------|
| **暂停/继续** | 任务卡住，需要你补充指令 |
| **另存为应用** | 跑通后固化流程，下次填参复用 |
| **再跑一次** | 失败后修复问题，重新执行 |
| **查看工作流** | 看 DAG 图和每个子任务的执行详情 |
| **批量操作** | 多条任务统一暂停/启动/删除 |

## 要点

- 项目团队 6 个角色（方案/计划/开发/审查/调试/验收）各司其职，比丢给一个全能角色产出质量高得多
- 任务中心是"跨会话看板"——Plan 发起的任务自动出现在这里，不需要手动切换
- 跑通后存为应用 = 下一次填参就能再造一个完整项目
