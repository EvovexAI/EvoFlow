# 案例七：总控智能体模式——多角色自动编排

> **场景：你要做一个"全栈项目开发"，需要不同角色分工——方案、编码、测试、审查、部署。**
>
> 总控智能体模式自动拆解任务、分配不同 AI 角色去干、监督进度、出错了自动调整方案。它就像是你的"技术总监"——你告诉它"做这个"，它自己安排人干。

## 操作步骤

### 1. 创建子智能体

先准备好你要调度的角色（参考[案例二](create-agent.md)）：

| 角色 | 职责 | 模型 |
|------|------|------|
| `project-architect` | 系统架构设计、技术选型、方案文档 | GPT-4（高思考预算） |
| `project-implementer` | 编码实现、代码落地 | Claude（编码能力强） |
| `project-qa` | 测试用例编写、测试执行与验收 | Claude（测试经验丰富） |
| `project-reviewer` | 代码审查、质量把控 | GPT-4（审查严格） |

你也可以直接用预设角色（系统内置的 `project-*` 系列，开箱即用）。

### 2. 切换到总控智能体模式

在对话页面的模式切换中，选择 **「总控智能体」** 模式。

### 3. 描述任务

```
帮我做一个全栈博客项目，技术栈 FastAPI + React + SQLite。
让 architect 先出设计方案，implementer 编码，
tester 写测试，reviewer 最后审查代码质量。
```

### 4. 总控智能体自动执行

总控智能体会自动：

1. **拆解任务**：把"全栈博客项目"拆成小程序（方案 → 后端 API → 前端 → 测试 → 审查）
2. **分配角色**：方案 → architect，编码 → implementer 等
3. **监督进度**：每个子任务完成后，总控智能体检查结果再派下一个
4. **异常处理**：如果某步骤失败，自动分析原因并调整方案重试

### 5. 查看结果

所有子任务完成后，总控智能体输出整合后的最终结果。你可以在任务中心看到完整的 DAG 工作流。

## 进阶：总控智能体 + 智能体员工

如果任务需要长期值守（比如"持续开发新功能"），可以：

1. 用总控智能体模式跑通一次完整的开发流程
2. 把流程另存为应用
3. 雇佣一个"项目管理员"员工，职责是"每周一运行项目开发应用，跟进开发进度"

这样总控智能体负责"编排"，员工负责"值班"——形成完整的生产力闭环。

## 与智能体员工的关系

| 维度 | 总控智能体模式 | 智能体员工 |
|------|----------------|-----------|
| 本质 | 一次性的多角色任务编排 | 长期的定时值班岗位 |
| 触发 | 你说"开始"它就干 | 到点自动上岗 |
| 角色 | 调度多个子智能体 | 单个岗位专注职责 |
| 产出 | 一次性交付物 | 持续巡检 / 周期性报告 |
| 组合 | 跑通后存为应用 | 调度应用执行 |

## 要点

- 子智能体分工越清晰，总控智能体产出质量越高——角色就是"专业的人干专业的事"
- 预设角色（`project-*` 系列）可以直接用，不必自己创建
- 总控智能体跑通全流程后，别忘了「另存为应用」——下次填参就能再造一个项目
- 如果任务不需要多角色协作，Plan 模式就够了，不必上总控智能体
