# 案例六：数据分析师每周出报表

> **场景：你每周需要分析项目数据，出一份带图表的数据分析报告。**
>
> 雇一个"数据分析师"员工，每周五自动分析数据、生成含图表的 HTML 报告，跑完直接交报告，不用你审批。

## 操作步骤

### 1. 创建数据分析角色

在「智能体」页创建角色，人设示例：

```
你是一个专业的数据分析师，精通 Python 数据分析（Pandas、Matplotlib、Plotly）。
分析时先理解数据结构和业务含义，再选择合适的分析方法。
输出 HTML 报告，包含可视化图表、关键发现和 actionable 建议。
```

配好 `deep-research` 技能（用于联网搜索行业数据对比）和 `terminal` 工具（运行 Python 脚本）。

### 2. 雇佣为员工

| 字段 | 填写示例 |
|------|----------|
| **岗位名称** | 数据分析师 |
| **职责** | 每周五下午 3 点，分析本周项目数据，生成 HTML 分析报告，包含图表和关键发现 |
| **管辖工作空间** | 项目数据目录 |
| **自主权** | **全自动**（跑完直接交报告，不用你审批） |
| **上岗节奏** | 每周五 15:00 |

### 3. 报告交付

每周五下午：

1. 员工自动上岗，读取本周数据
2. 执行 Python 分析脚本，生成图表
3. 输出 HTML 报告到工作空间
4. 在「任务中心」的「待确认」列出现报告项

你打开报告，确认即可。

### 4. 进阶：员工 + 应用中心

如果报告格式固定，还可以把分析流程存为应用：

1. 让员工先跑通一次分析流程
2. 把流程"另存为应用"→ 参数化（`数据时间范围`、`分析维度`、`输出格式`）
3. 以后员工可以调用这个应用，填参运行

这样员工负责"到点触发"，应用负责"标准化执行"——各司其职。

## 员工 + 应用中心组合模式

| 场景 | 员工负责 | 应用负责 |
|------|----------|----------|
| 每周数据报告 | 定时触发 + 确认数据源 | 标准化分析流程 + 图表生成 |
| 每日巡检 | 定时触发 + 异常判断 | 标准检查流程 + 报告生成 |
| 竞品跟踪 | 定时触发 + 选择调研对象 | 标准调研流程 + 对比分析 |

## 要点

- 全自动自主权适合"流程稳定、不需要人为判断"的场景
- 全自动模式下，员工跑完直接交报告到「待确认」列，你验收即可
- 如果报告需要联网搜索行业数据，角色要配 `deep-research` 技能和 `web_search` 工具
- 报告 HTML 文件可以直接在浏览器打开，也可以分享给团队
