# Plan 模式（任务协作）

> **什么时候用 Plan 模式？**
>
> 当你有一件事需要分几步做，而且"先对好方案再动手"更稳妥的时候：
>
> - **先调研、再写方案、最后开发代码**：比如"调研 3 个前端框架，选一个写 demo，再写对比报告"——AI 先出计划，你确认后再执行
> - **写一份带数据图表的周报**：先让 AI 列出"搜集数据→做图表→写报告"三步计划，每步出结果你确认
> - **部署一个服务到服务器**：先规划"环境检查→配置→部署→验证"四步，每一步你验收完再走下一步
>
> Plan 模式的核心是：**先定稿计划再授权执行**，避免 AI 一上来就瞎干。跟[目标智能体](goal-agent.md)的区别是：Plan 是"你说好再做"，目标是"你说了它自己跑"。
>
> 计划落库后可以在[任务中心](../tasks/task-center.md)跨会话查看和管理；步骤可指定[项目团队](preset-roles.md)角色（如 `project-architect`、`project-implementer`）作为执行人。

复杂多步骤任务：先澄清并调用 `plan` **落库** → 输入框上方确认条授权 → Supervisor 按步骤图执行。计划正文在**主任务表**，不是工作区里的 `plan.md`。

> **入口**：聊天输入框底部模式药丸选 **Plan**（Ask / Agent / Plan）。  
> **不是**顶栏「任务规划」场景名；也**不是** [目标 Agent](goal-agent.md)（Goal 是另一套长程自驱）。

---

## 和其它能力怎么选

| 你想做的事 | 用 |
|----------|-----|
| 单会话多步、先对齐再授权执行 | **Plan**（本文） |
| 跨会话看板、暂停/验收多任务 | [任务中心](../tasks/task-center.md)（Plan 落库后会出现在来源「对话」） |
| 固定流程填参再跑 | [应用中心](../configuration/app-center.md)（确认条可「另存为应用」） |
| 到点重复固定 Prompt | [自动化](../tasks/scheduled-tasks.md) |
| 开放长任务自驱 | [目标 Agent](goal-agent.md)（聊天 Goal，与 Plan 无关） |

项目团队（`project-*`）是 Plan 步骤上的**执行人**，不是第二套 Plan，见 [项目级 Plan 流程](project-team-plan-workflow.md)。

## 如何开启

1. 打开实时聊天
2. 底部模式选 **Plan**（Toast：已切换为：Plan）
3. 输入框 placeholder 类似：描述多步骤目标；将先对齐需求再规划…
4. 发送需求

选 Plan 会联动协作规划阶段（`is_plan_mode` + planning）。

| 模式 | 适合 |
|------|------|
| **Ask** | 问答、少副作用 |
| **Agent** | 直接干活 |
| **Plan** | 先定稿计划，再「开始执行」 |

---

## 完整流程（按界面）

1. **发需求** → 助手可摸底、澄清，再调用 **`plan`** 写入任务表  
2. **消息区**出现 plan 工具摘要（目标、步骤数）  
3. **输入框上方确认条**出现，例如：  
   - kicker：**计划已定稿**  
   - 状态：**待授权开始执行**  
   - 摘要：N 个步骤 / 子任务  
4. 点 **查看计划** → 弹窗标题 **任务计划**（目标 / 执行步骤 / 整体验收）  
5. **要改计划**：在下方输入框说明修改意见（**没有**「修改计划」独立按钮）  
6. 点 **开始执行** → 授权执行 → 自动打开右侧 **工作流** DAG  
7. 可选：**另存为应用**、**不再提示**

### 确认条按钮

| 按钮 | 作用 |
|------|------|
| **查看计划** | 打开「任务计划」弹窗 |
| **开始执行** | 授权并启动（进行中可能显示「启动中…」） |
| **另存为应用** | 沉淀到应用中心 |
| **不再提示** | 收起本条确认提示 |

勿依赖助手气泡里的「请点击开始执行」文案——以确认条为准。

## 工作流面板（右侧）

| 时机 | 界面行为 |
|------|----------|
| plan 已定稿、**尚未**点开始执行 | 协作侧栏**故意不展示**子任务列表（避免未授权就当已开工） |
| **开始执行**之后 | 自动打开右侧 **工作流**；顶栏可「显示/隐藏子任务工作流」 |
| 点击节点 | 查看子任务对话与执行详情 |

全屏同款 DAG：任务详情里 **查看工作流** → `#/workflow/:id`（见 [任务中心](../tasks/task-center.md)）。

授权后**不会**自动跳转任务中心；需要时可手动打开 `#/tasks`，来源 Tab 选 **对话**。

---

## 子任务何时创建、何时算跑起来

| 时机 | 行为 |
|------|------|
| **`plan` 成功** | 后端按步骤同步创建/更新子任务（一般不必再 `create_subtasks`） |
| **列表可见** | **授权后**在工作流面板看到节点推进（定稿未授权时侧栏隐藏） |
| **开始跑** | 用户点「开始执行」→ 授权 → `start_execution` / 派发 → 子任务 `executing` |
| **算完成** | Worker 上报结果后后端标完成；勿因委派返回 `ok` 就当侧栏已完成 |

步骤间 `depends_on` 在上游完成后由后端自动跟进下一波。

## Plan 与 TodoList

| | **Plan（任务协作）** | **TodoList（`write_todos`）** |
|--|----------------------|-------------------------------|
| 用途 | 定稿多步计划 + 授权 + Supervisor DAG | 会话内轻量待办 |
| 落库 | 主任务表 | 中间件待办状态 |
| UI | 确认条 + 工作流 + 任务中心 | 一般不走「开始执行」确认条 |

简单任务助手可不建 Plan，直接在 Agent 模式做。

## 规划阶段保护（PlanGuard，简要）

在计划未授权执行前，系统会限制高副作用工具，保留读文件、澄清、`list_agents`、`plan`、调度类等。点「开始执行」进入执行阶段后限制放宽。细节见系统文档 / harness 实现。

## 常见问题

**Q：为什么 plan 之后看不到子任务？**  
定稿未授权时侧栏故意隐藏。点 **开始执行** 后看右侧 **工作流**。

**Q：没有「修改计划」按钮？**  
在输入框说明修改意见即可，助手会再调 `plan` 更新任务表。

**Q：计划正文在哪？**  
确认条 **查看计划**，或任务中心详情 / API 任务表。

**Q：和「目标」有何区别？**  
目标 = 聊天 Goal 长程自驱；Plan = 先定稿再授权的多步协作。入口都在聊天，但是不同模式。

**Q：刷新后确认条没了？**  
强刷后应从任务快照恢复；若 plan 未成功落库则不会出现确认条。

**Q：项目团队怎么配合？**  
见 [项目级 Plan 流程](project-team-plan-workflow.md)；步骤上指定 `project-*` 执行人。
