# 自动化操作指南

> **比如你想让 EvoFlow 每天自动做这些事**：
>
> - **每天上午 9 点**：检查服务器健康，异常就告警
> - **每周五下午 5 点**：汇总本周项目进展，生成周报推给飞书
> - **每月 1 号**：跑一次数据汇总报告
>
> 配一个 cron 表达式，写清楚要 AI 执行的指令，到点自动跑。结果可以推送到飞书、Slack 等渠道。
>
> 自动化与「任务中心」互补：自动化是定时跑固定指令，任务中心是多步骤复杂任务的看板。

自动化是 **到点重复（或一次性）** 触发：你定时间规则，到点系统起一轮 Agent 会话执行固定 Prompt，结果可选推送到已配置的消息渠道。适合日报、巡检、定期汇总等「机械重复」工作。

侧栏入口：**自动化**（`#/cron`）。

---

## 一、何时用（别和任务中心 / 目标混）

| 你想做的事 | 推荐路径 |
|----------|---------|
| **到点重复**固定指令（每天 9 点出日报） | **自动化**（本文） |
| 复杂多角色、要看板观测的一次性工程 | [任务中心](task-center.md) |
| 流程已固定、只换参数再跑 | [应用中心](../configuration/app-center.md) |
| 岗位定时巡检、审批交班 | [智能体员工](../configuration/smart-employees.md) |
| 单会话边聊边拆步 | [Plan 模式](../chat/plan-mode.md)（聊天顶栏） |
| 开放长任务、聊天里自驱多轮 | [目标 Agent](../chat/goal-agent.md)（**实时聊天 Goal 模式**，不是任务中心，也不是本页） |

自动化特征：**频率固定、内容相对稳定、不需要人盯**。

## 二、页面布局

| 区域 | 文案 / 内容 |
|------|-------------|
| 标题 | **自动化** |
| 副标题 | 自动化调度管理 — 创建、监控和管理周期性 / 一次性任务 |
| 主按钮 | **＋ 新建任务** |
| 状态栏 | **调度器运行中 / 未启动 / 异常** · **活跃** · **已暂停** · **总计** |
| 空态 | 暂无自动化 · 点击「新建任务」创建… |

### 任务卡片

- 名称、执行频率摘要、任务描述（前几行）
- 徽标：**运行中** / **已暂停** / **一次性**
- 开启推送时显示渠道标签（如「飞书默认群」），不是手填 chat_id
- 操作：**▶ 运行** · **⏸⏸ 暂停** · **↻ 恢复** · **📋 历史** · **✎ 编辑** · **✕ 删除**

> 列表卡片一般**不显示**「下次执行时间」；预览在创建/编辑弹窗的调度摘要里。

## 三、创建任务（界面）

点 **＋ 新建任务**，弹窗为三个 Tab。

### Tab 1 · 基本信息

| 字段 | 必填 | 说明 |
|------|------|------|
| **任务名称** | 是 | 如：每日健康检查 |
| **任务描述 (Prompt)** | 是 | 每次发给 Agent 的指令，写清做什么、产出什么 |

Prompt 宜具体，少寒暄。例：「汇总昨日本仓库 PR 与 Issue 变化，生成 Markdown 日报」。

### Tab 2 · 调度配置

| 模式 Tab | 说明 |
|----------|------|
| **每小时** | 每小时第 MM 分触发 |
| **每天** | 选时刻（含上下午） |
| **每周** | 星期 chips + 时刻 |
| **一次性** | 日期时间；快捷：**现在** / **+1小时** / **明早 9:00** / **今晚 22:00** |
| **自定义 Cron** | 五字段：`分 时 日 月 星期` |

底部有调度摘要预览（如「每天 09:00 执行」）。

### Tab 3 · 高级设置

| 选项 | 默认 | 说明 |
|------|------|------|
| **工作目录** | 空 | 可「选择文件夹」；空则用 Gateway 默认 |
| **任务超时时间（分钟）** | **30** | 单次上限，常见范围 1–1440 |
| **完成后推送结果** | 关 | 打开后选 **推送渠道**（需先配好消息渠道） |
| **记忆 · 启用** | 关 | 见下方「记忆 vs 线程」 |

无可用渠道时提示：**暂无可用的推送渠道**。

按钮：**取消** · **创建任务**（编辑时为 **保存修改**）。

### 对话创建（可选）

在聊天里说明「帮我建一个每天 9 点…的自动化」，确认后创建，与 GUI 等价。

## 四、运行、暂停与历史

| 操作 | 说明 |
|------|------|
| **▶ 运行** | 立即手动触发一次（不改调度） |
| **暂停 / 恢复** | 控制是否参与到点调度 |
| **历史** | 侧栏/面板：**执行历史 — {任务名}** |
| **编辑 / 删除** | 改配置或移除（删除需确认） |

历史行常见：触发方式、耗时、thread 摘要；有会话信息时可 **在聊天中打开**。

执行记录在 Gateway 数据（SQLite）中，不是旧版 `*_history.jsonl` 为主路径。

## 五、推送结果

1. 先在设置 / 消息渠道配好飞书等（见 [飞书集成](../integration/feishu-integration.md)、[IM 渠道](../integration/im-channels.md)）
2. 编辑自动化 → **完成后推送结果** → 选 **推送渠道**
3. 跑一次（或等下次到点）验证渠道能收到

主路径是渠道单选，不是在表单里手填 chat_id（高级/遗留配置见调度参考）。

## 六、记忆 vs 线程

| 概念 | 面板 | 含义 |
|------|------|------|
| **记忆 · 启用** | 有开关 | `memory_enabled`：是否带记忆能力 |
| **LangGraph 线程 sticky/fresh** | **无 UI** | 默认每次 **fresh** 新线程；要跨次复用同一线程需 TOML/API 设 `langgraph_thread_mode=sticky` |

**记忆开关 ≠ 自动 sticky。** 多数日报类任务保持记忆关闭、每次独立更干净。

深读：[自动化调度参考](automation-scheduler.md)。

## 七、常见问题

**Q：到点没跑？**  
卡片是否 **运行中**（非已暂停）；Gateway 是否在跑；全局调度是否被 `EVOFLOW_AUTOMATION_SCHEDULER=0` 关掉；cron 时区与本机是否一致；并发是否打满。

**Q：状态栏一直「调度器未启动」？**  
面板状态指示可能未实时拉 Gateway；以任务是否到点执行、Gateway 日志为准。也可用调度参考里的 API 自查。

**Q：一次性任务跑完后？**  
通常不再按周期触发；徽标仍可能显示「一次性」。需要重复请改成每天/每周或再建一条。

**Q：超时默认多少？**  
界面默认 **30 分钟**（不是 60）。

**Q：和任务中心有何区别？**  
自动化 = 定时跑固定 Prompt；任务中心 = 多步骤协作任务的看板与观测。二者入口、数据模型都不同。

**Q：目标 Agent 在哪？**  
实时聊天的 **Goal** 模式，见 [目标 Agent](../chat/goal-agent.md)。不在任务中心，也不在本页。
