# 面板设置

> **设置弹窗是 EvoFlow 的"控制面板"——7 个 Tab 管所有偏好，每个 Tab 里的每个选项都有它的作用。**
>
> 左下角齿轮图标打开，所有变更自动保存，按实例（本机/Docker/远程）独立存储。

---

## 弹窗结构

设置弹窗采用「左侧导航 + 右侧面板」布局，共 7 个 Tab。每个 Tab 按需加载，切换不重刷。URL 带 hash（`#/settings?tab=xxx`），可直接外链到具体 Tab。

---

## 一、通用 Tab

控制 EvoFlow 的"长相"和"安全策略"。

### 主题与外观

| 选项 | 可选值 | 默认值 | 作用 |
|------|--------|--------|------|
| **主题模式** | 日间 / 夜间 / 跟随系统 | 跟随系统 | 切换界面深色浅色。选"跟随系统"则随操作系统自动切换 |
| **字体大小** | 小 / 中 / 大 / 超大 | 中 | 调整聊天和菜单的文字大小 |
| **界面语言** | 中文 / 英文 | 跟随系统 | 界面文字的语言。AI 回复的语言默认跟随此设置 |

> 这些只影响界面外观，不影响功能。不改也能正常用。

### 工具审批策略

控制 AI 在调用高风险工具时是否要你确认：

| 策略 | 作用 | 什么时候用 |
|------|------|-----------|
| **默认询问** | AI 调用 terminal、delete、网络请求等高风险工具时，弹出审批条，你点"同意"才执行 | 日常推荐，安全又可控 |
| **自动批准** | 所有工具调用直接执行，不询问 | 个人本地环境、你信任 AI 不会乱来的时候 |
| **仅会话内授权** | 同一会话内首次调用某个工具时询问，同意后后续同名工具自动通过 | 长任务场景，省得每次都要点同意 |

> 也可以对**单个工具**单独覆盖策略。比如你希望 terminal 每次都问，但 delete 自动批准——可以单独设置。

### 数据目录

| 选项 | 作用 |
|------|------|
| **打开数据目录** | 在文件管理器中打开 `~/.evoflow/`，直接查看会话、记忆、配置文件 |
| **清理缓存** | 清理临时缓存文件。长期不清理可能占磁盘空间 |

> 数据目录存储了所有会话历史、记忆、自动化配置等。一般不需要手动操作，但如果你要备份或迁移数据，可以从这里找到原始文件。

## 二、模型 Tab

快速切换 AI 对话用的模型。

| 选项 | 作用 |
|------|------|
| **默认对话模型** | 下拉选择已配置的模型。切换后后续对话立即使用新模型 |
| **连接状态** | 显示当前模型是否可用（已连接 / 连接失败） |

> **这里只能快速切换已有模型。** 如果要添加新的模型厂商（比如新增通义千问）、调整温度/思考预算等参数，请去主菜单「模型配置」做完整配置。
>
> 比如：你现在用 DeepSeek V4，觉得回复太慢，想换 GPT-5.6 试试——在这里下拉切换就行，不用去翻完整配置页。

---

## 三、环境变量 Tab

集中管理所有 API Key 和系统变量，AI 跑命令、脚本、子代理时自动引用。

| 操作 | 作用 |
|------|------|
| **添加变量** | 以 `KEY=VALUE` 格式添加环境变量，如 `OPENAI_API_KEY=sk-xxx` |
| **掩码显示** | 敏感值在列表中显示为 `sk-•••••`，防止窥屏 |
| **编辑/删除** | 修改或移除已添加的变量 |

> 写入后**不需要重启**，对以下场景自动生效：
> - AI 执行 terminal 命令时
> - 技能脚本运行时
> - 子代理执行任务时
> - MCP 工具连接时
>
> **不配的话**：每次用新工具/服务都要单独找密钥，很麻烦。而且同一个密钥可能在多个地方需要（比如模型 API 和 MCP 工具都要用 OpenAI Key），统一管理后改一次全局生效。

**常见用法：**

```
OPENAI_API_KEY=sk-xxxxx
ANTHROPIC_API_KEY=sk-ant-xxxxx
DEEPSEEK_API_KEY=sk-xxxxx
HTTP_PROXY=http://127.0.0.1:7890
GIT_COMMITTER_NAME=Alice
```

## 四、IM 通信 Tab

配置飞书、Slack、Telegram 等 IM 渠道的凭证，让自动化推送、智能体员工审批等通知能主动推送到你的聊天工具。

| 选项 | 作用 |
|------|------|
| **渠道类型** | 选飞书 / Slack / Telegram / Discord 等 |
| **凭证配置** | 填写对应渠道的 Bot Token、App ID、Secret 等 |
| **连接状态** | 显示渠道是否配置成功 |

> **不配的话**：自动化任务跑完了，结果只能留在 EvoFlow 面板里看，不会主动推送到你的飞书或 Slack。智能体员工的待审批事项也需要你打开 EvoFlow 才能看到。
>
> 具体每个渠道的配置步骤，见 [IM 消息渠道配置](../integration/im-channels.md)。

## 五、记忆 Tab

快速浏览 AI 记住了你的哪些信息，也可以手动编辑或删除。

| 选项 | 作用 |
|------|------|
| **全局记忆** | 主智能体记住的关于你的信息，如名字、工作背景、偏好 |
| **各 Agent 记忆** | 下拉选择具体角色，查看该角色独立记住的信息 |
| **编辑/删除** | 修改或删除某条记忆。比如你之前告诉 AI"我叫小明"，它记住了，想改可以在这里改 |

> **不配也行**：默认自动记忆，AI 会在对话中自动学习你的偏好。但如果你发现 AI 记住了不该记的东西（比如你随口说了一句不想被记住的），可以来这里删掉。
>
> 每个自定义角色有独立的记忆文件，互不污染。记忆的详细机制见 [记忆管理](memory-management.md)。

## 六、快捷键 Tab

把常用操作绑定成键盘快捷键，不用每次点鼠标。

| 预置动作 | 说明 |
|---------|------|
| **发送消息** | 发送当前输入框内容 |
| **新会话** | 开启一个新会话 |
| **停止生成** | 停止 AI 正在回复的内容 |
| **打开命令面板** | 打开快捷指令面板 |
| **切换 Agent** | 在角色之间切换 |
| **切换模式** | 在 Ask/Agent/Plan 模式之间切换 |

> 每个动作可以绑定一个快捷键组合（如 `Ctrl+Shift+P`）。如果跟系统级快捷键冲突，会有告警提示。
>
> **不配也行**：所有操作都能用鼠标完成。但如果你每天用 EvoFlow 很长时间，绑几个常用快捷键能省不少时间。
>
> 一键恢复默认可以回到初始设置。

## 七、关于 Tab

查看 EvoFlow 的版本信息。

| 选项 | 作用 |
|------|------|
| **版本号** | 当前安装的 EvoFlow 版本 |
| **检查更新** | 联网检查是否有新版本 |
| **许可证** | 查看当前使用的许可证信息 |
| **致谢** | 开源组件和贡献者列表 |

> 升级客户端前可以先来看看有没有新版本。

## 多实例支持

EvoFlow 支持同时管理**本机**、**Docker**、**远程**三类实例。

| 特性 | 说明 |
|------|------|
| **设置独立存储** | 本机配的快捷键、环境变量、主题等，和远程实例互不冲突 |
| **实例切换** | 左侧菜单顶部切换器，随时切到另一个实例 |
| **远程实例添加** | 见 [EvoFlow 桌面端使用指南](evopanel-guide.md) |

> 比如你在公司电脑上配了深色主题 + 一组快捷键，回到家连远程服务器，远程实例有自己的设置，互不影响。
