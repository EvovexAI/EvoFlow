# 应用中心

> **应用中心就是把你"已经跑通的工作流"固化下来，以后填参数就能再跑一遍。**
>
> ---
>
> **场景一：你在 Plan 模式里跑通了一个"竞品调研"任务**
>
> 你第一次让 AI 做"调研 3 家国产 AI Agent 平台的定价、功能、最近动态"——AI 先出计划，你确认后它执行，最后出了一份对比表格。结果很好。
>
> 但问题是：下周你可能还要再调研一遍，下个月也要。每次从零跟 AI 说一遍太麻烦。
>
> **应用中心帮你解决这个问题：**
> 1. 在 Plan 确认条点"**另存为应用**"
> 2. 系统自动把刚才的流程变成"应用"，你给它起个名叫"竞品调研"
> 3. 你定义两个参数：`调研目标`（比如"AI Agent 平台"）和 `对比维度`（比如"定价、功能、评价"）
> 4. 点"发布"，以后在应用中心找到它，填参数，点运行，自动出结果
>
> 从"每次从零对话"变成"选应用→填参数→跑"。
>
> ---
>
> **场景二：你有一个"每周自动出周报"的固定流程**
>
> 你已经在 Plan 模式里调好了周报生成的步骤：搜集本周项目动态 → 按项目归类 → 生成 Markdown 周报 → 推送到飞书。
>
> 把这个流程存成"周报生成"应用，参数设为`周次`和`项目名称`。以后每周五，你打开应用中心，选这个应用，填上"第 32 周"和"EvoFlow 项目"，点运行，周报自动生成。
>
> ---
>
> **应用中心适合什么？**
> - 报告生成（竞品分析、周报、调研报告）
> - 代码模板（新项目脚手架、API 代码生成）
> - 数据处理（日志分析、数据清洗、报表生成）
> - 审查清单（代码审查、发布检查、安全审计）
>
> **跟其他功能的区别：**
> - 实时聊天/Plan：探索阶段、一次性复杂协作
> - 任务中心：看某次多 Agent 任务的执行与子步骤
> - 自动化：到点重复跑固定 prompt
> - **应用中心：流程已固定，只换参数反复开跑**
>
> 侧栏入口：**应用中心**（`#/apps`）。

把已经跑通的多步工作流存成**应用**：以后填参数就能再跑，不必每次从零对话。侧栏入口：**应用中心**（`#/apps`）。

帮助文案：新建 → 画布编排 → 发布 → 卡片「···」→ **填参运行**。

---

## 和任务中心 / 聊天有何不同

| 能力 | 适合 |
|------|------|
| **实时聊天 / Plan** | 探索、一次性复杂协作 |
| **任务中心** | 看某次多 Agent 任务的执行与子步骤 |
| **自动化** | 到点重复跑固定 prompt |
| **应用中心**（本文） | 流程已固定，只换参数反复开跑（竞品分析、周报、审查清单等） |

---

## 路由一览

| 路由 | 用途 |
|------|------|
| `#/apps` | 应用列表 |
| `#/apps/:id` | 画布编排（沉浸，隐藏全局侧栏） |
| `#/apps/:id/run` | 外部填参运行 |
| `#/apps/:id/history` | 运行历史与结果 |

---

## 列表页

页头提示：① 画布编排 → ② 发布 → ③ ··· → 填参运行。

| 控件 | 说明 |
|------|------|
| 筛选 | **活跃** / **已归档** / **全部** |
| 搜索 | 按名称过滤 |
| **新建应用 ▾** | **空白应用** / **从模板创建** / **从任务沉淀** |
| 卡片 | 进入画布 `#/apps/:id` |
| 状态徽章 | **草稿** / **已发布** / **已归档** |

### 已发布卡片常用操作

| 操作 | 说明 |
|------|------|
| **▶ 运行** / **填参运行** | 进入 `#/apps/:id/run` |
| **编辑流程** | 回画布 |
| **编辑信息** | 改名称等元信息 |
| **归档** / **删除** | 离开活跃列表或永久删除 |

空态可：**创建空白应用** 或 **从模板创建**。

## 画布编排 `#/apps/:id`

工具栏（与界面一致）：

| 控件 | 说明 |
|------|------|
| 返回应用列表 | 回 `#/apps` |
| 应用名称 | 可直接改 |
| 状态徽章 | 草稿 / 已发布 / 已归档 |
| **节点** | 节点库与节点配置 |
| **参数** | 应用设置与运行参数 |
| **历史** | 跳转运行历史 |
| **版本** | 版本快照 |
| **API** | Key 与 OpenAPI（须**已发布**） |
| **保存** | 存为草稿 |
| **发布** | 列表填参运行与 API 的前置条件 |
| **调试** | 画布内填参试跑 |

### 应用设置（参数侧栏 / 模态）

| 字段 | 说明 |
|------|------|
| **工作流目标** | 应用要达成什么 |
| **描述** | 列表与分享说明 |
| **执行方式** | 按步骤自动跑完 / 先生成计划再确认（监督模式） |
| **调度触发时自动执行** | 仅无人场景勾选 |
| **运行参数** | 英文标识符表单字段，供填参页使用 |

未发布时，对外填参运行与 API 不可用或受限——先 **发布**。

---

## 填参运行 `#/apps/:id/run`

| 控件 | 说明 |
|------|------|
| ← 应用列表 | 返回 |
| **编辑流程** | 回画布 |
| 参数表单 | 按应用定义的运行参数填写 |
| **运行** | 启动 |
| **执行中** / **暂停** / **继续** / **取消** | 运行态控制 |

跑完可在 **历史** 页回看同一次结果；任务侧也可「再跑一次」跳到填参页。

---

## 运行历史 `#/apps/:id/history`

- 历次运行列表与结果摘要
- 便于对照参数、排查失败、复制再跑

---

## 推荐用法

1. 先在聊天或任务中心把流程跑通一次  
2. **从任务沉淀** 或空白画布整理成应用  
3. 配好 **运行参数** → **保存** → **发布**  
4. 日常只走 **填参运行**，少改画布  

与 [智能体员工](smart-employees.md) 的区别：员工是定时岗位巡检；应用是「可重复的工作流产品」。

---

## 常见问题

**Q：点运行没反应 / 不能填参？**  
确认应用状态为 **已发布**，且已定义运行参数（可为空参数直接跑）。

**Q：画布里调试成功，列表却跑不了？**  
调试可在草稿上试；列表「填参运行」一般要求已发布。

**Q：API 入口灰掉？**  
先发布；再在画布 **API** 中查看 Key / OpenAPI。

**Q：和自动化调度怎么配合？**  
应用可被调度触发；勾选「调度触发时自动执行」时注意无人场景与审批策略。
