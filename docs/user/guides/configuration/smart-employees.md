# 智能体员工

> **智能体员工就是让 AI 从"你问它答"变成"主动上班干活"。**
>
> ---
>
> **场景一：你有一个"服务器巡检"的固定需求**
>
> 你需要每天检查 3 个服务（gateway、langgraph、postgres）是否健康，有异常就告警，正常就写个巡检报告。
>
> **以前**：每天手动登录服务器，敲命令检查，写报告……重复劳动。
>
> **用智能体员工**：
> 1. 先在"**智能体**"页创建或选一个角色，比如"运维助手"——给它配好 terminal 工具和飞书推送权限
> 2. 到"**智能体员工**"页，点"雇佣员工"
> 3. 配置：
>    - **岗位名称**：服务器巡检
>    - **职责**：每 30 分钟检查一次 gateway/langgraph/postgres 的健康状态，异常时推送飞书告警，下班前写巡检报告
>    - **管辖工作空间**：选一个目录，报告会写到那里
>    - **自主权**：选"谨慎型"——所有事项先问你再做（推荐上手）
>    - **上岗节奏**：每 30 分钟
>    - **上班时间**：09:00–20:00
> 4. 点"确认上岗"
>
> **之后每天**：
> - 09:00：员工自动上岗，开始巡检
> - 09:30：第一轮完成，报告写到工作空间
> - 遇到异常：员工先暂停，给你发"待审批"——你同意告警，它才推飞书
> - 20:00：写交班报告，下班
>
> 你只需要在"**待审批**"Tab 里看看有没有事项，同意或驳回即可。
>
> ---
>
> **场景二：你家有个"代码审查员"**
>
> 你有一个"代码审查"角色，专门做 PR 审查。你把它雇成员工：
> - **职责**：每 2 小时检查一次项目仓库的新提交，执行代码审查，输出审查意见
> - **自主权**：选"平衡型"——常规审查自动完成，遇到重大设计问题才问你
>
> 它会在后台自动审查代码，审查结果写到工作空间，你只需要在"工作项"看板里查看结果。
>
> ---
>
> **场景三：你有一个"数据分析师"每周出报表**
>
> 你有一个"数据分析师"角色，把它雇成员工：
> - **职责**：每周五下午 3 点，分析本周数据，生成数据报表
> - **上岗节奏**：每周五 15:00
> - **自主权**：选"全自动"——跑完直接交报告，不用你审批
>
> 每周五下午，报告自动生成，你打开看板验收就行。
>
> ---
>
> **智能体员工 vs 自动化的核心区别：**
>
> | 维度 | 自动化 | 智能体员工 |
> |------|--------|-----------|
> | 本质 | 到点跑一条固定 prompt | 一个有职责、有审批、有交班的岗位 |
> | 灵活度 | 每次都执行同样的指令 | 每次根据工作区当前状态自主决策 |
> | 审批 | 无 | 开工前审批、交工后验收 |
> | 交班 | 无 | 写交班报告、记录工作轨迹 |
> | 组织关系 | 无 | 有部门、上下级、看板 |
>
> **一句话总结**：自动化是"到点跑固定脚本"，员工是"雇了个 AI 员工上班"。
>
> 侧栏入口：**智能体员工**（`#/proactive`）。

让已有**智能体**按岗位定时上岗：巡检工作区、建工作项、等你拍板、交班写报告。侧栏入口：**智能体员工**（`#/proactive`）。

> **和「智能体」页不是一回事**  
> - **智能体**（`#/expert`）：配置人设、工具、技能——「会什么」  
> - **智能体员工**：给智能体加岗位合同（职责、工作区、心跳、审批）——「什么时候主动干、干到哪」  
> 雇佣前须先在「智能体」页有对应 Agent（可用 [预设角色](../chat/preset-roles.md) 或 [自行创建](agent-management.md)）。

帮助面板里本页标题为「主动智能」：创建岗位 → 配置职责 → 自动运转。

## 前置条件

1. 已配置可用对话模型（侧栏「模型配置」）
2. 「智能体」页已有目标 Agent
3. 对话侧栏已添加本机**工作空间**目录（雇佣时要从历史里选「管辖工作空间」）
4. （可选）[飞书集成](../integration/feishu-integration.md)：待审批可推到飞书卡片

全局值班由面板「值班台」开关控制；若引擎被环境关闭（如 `EVOFLOW_PROACTIVE_SCHEDULER=0`），页面会提示无法连接值班服务。

---

## 5 分钟上手

1. 侧栏点 **智能体员工**
2. 值班台点 **开启值班**（状态变为「全员值班中」）
3. 点 **雇佣员工**，填写：
   - **绑定智能体**：选已有 `agent_code`
   - **岗位名称**、**职责**（每行一条）
   - **管辖工作空间**（必选）
   - **自主权**：建议先选 **谨慎型**（所有事项须你同意）
   - **上岗节奏**：如每 30 分钟 / 每小时
4. 点 **确认上岗**（也可先「保存草稿」）
5. 等下一次心跳，或卡片菜单点 **立即开工**（不受上班时间限制）
6. 有事项时到 **待审批** Tab：`Y` 同意 · `N` 驳回 · `J` 下一条

也可在「智能体」页对某角色点 **雇佣** / **值班台**，跳到同一套流程。

## 主界面导览

页头文案：**数字员工 · 智能体员工**——有事项等你拍板时优先处理；员工近况看「员工」，进度全貌看「工作项」。

| 区域 / Tab | 作用 |
|------------|------|
| **值班台** | 全局 **开启值班** / **结束值班** |
| **待审批** | 开工前需你拍板的事项（同意后才执行） |
| **员工** | 名册卡片：状态、近况、菜单操作 |
| **工作项** | 跳转全员看板 `#/proactive/board` |
| **组织架构** | 按部门 / 上下级看编制 |
| **健康** | 引擎与运行健康 |
| **归档** | 已归档员工；可重新上岗或彻底删除 |

### 常用路由

| 路由 | 页面 |
|------|------|
| `#/proactive` | 控制台 |
| `#/proactive/board` | 岗位工作项看板（可 `?role=<code>&hideDone=1`） |

## 雇佣与编辑字段

「雇佣上岗」与「编辑岗位」共用同一套表单。

| UI 标签 | 说明 |
|---------|------|
| **绑定智能体** | 底层 Agent；人设/工具仍回「智能体」页改 |
| **岗位名称** | 展示名 |
| **职责** | 每行一条；决定巡检时优先想什么 |
| **管辖工作空间** | 本机目录；方案/报告会写到该工作空间下 |
| **关注子路径** | 相对工作空间的子目录，缩小巡检范围 |
| **部门** / **直属上级** | 组织关系（高级区）；影响催办与上下级视图 |
| **工作/执行模型** | 空 = 跟随智能体默认模型 |
| **自主权** | 见下表 |
| **上岗节奏** | 心跳 RRULE：10/30 分钟、1/2/3 小时、每天 9:00 等 |
| **上班时间** | 默认约 09:00–20:00；窗外一般不自动心跳（**立即开工**仍可） |
| **工作方式** | **深度工作**（可调工具的 agent loop）/ **轻量工作**（单次 LLM） |

### 自主权三档

| 芯片文案 | 含义 |
|----------|------|
| **全自动** | 仅极高风险须你同意；其余自动执行 |
| **平衡型** | 中风险及以上须你同意；低风险可自动（常见默认） |
| **谨慎型** | 所有事项须你同意后才执行（推荐上手） |

### 状态

| 标签 | 含义 |
|------|------|
| **上班中** | 参与心跳与值班 |
| **已请假** | 暂停自动上岗 |
| **草稿** | 未确认上岗；可「确认上岗」 |
| **已归档** | 不出现在名册、不自动上班；历史保留 |

## 日常操作

| 操作 | 说明 |
|------|------|
| **立即开工** | 手动触发一轮值班心跳 |
| **派发任务** | 指定标题/说明派给该岗；打开工作轨迹跟进 |
| **请假** / **上班** | 暂停或恢复自动心跳 |
| **编辑岗位** | 改职责、节奏、工作区等 |
| **归档** | 离开名册；可在「归档」Tab **重新上岗** |
| **删除** | 去掉岗位编制（底层智能体仍在「智能体」页） |

同一员工若已有一轮在跑，再点开工会提示稍候。

## 工作项看板

列含义：

| 列 | 含义 |
|----|------|
| **待办** | 已建项、尚未执行 |
| **执行中** | 员工正在处理 |
| **待确认** | 员工已交工，等你确认（验收） |
| **已完成** | 闭环 |
| **失败** | 执行失败或未完成交班等 |

### 待审批 vs 待确认

| | **待审批**（控制台 Tab） | **待确认**（看板列） |
|--|--------------------------|----------------------|
| 时机 | **开工前**拍板 | **交工后**验收 |
| 动作 | 同意执行 / 驳回（需原因） | 确认完成或退回（以页面按钮为准） |
| 超时 | 超时项显示为「审批超时」 | — |

不要把「待确认」当成「同意执行」——前者是收工验收，后者是放行开工。

## 工作日志与交班

进入 `#/proactive/:code` 可看：

- 值班轮次、工作项列表
- 轮次阶段：上岗（check_in）→ 本轮小结（wrap_up）
- **工作轨迹**：本轮工具调用与对话过程
- 值班记忆、近 7 日成本 / 表现（若面板展示）

若出现 **自动交班·未完成** / **【交班汇报·未完成】**：

1. 打开该轮 **工作轨迹**，看是否中途中断、超时或未调用交班
2. 可再点 **立即开工** 让下一轮继续；不要只改职责文案碰运气
3. 交付文档是否落盘见下一节

## 交付文档写在哪

方案 / 报告 / 纪要类 Markdown 写到工作空间下：

```text
docs/roles/<agent_code>/<YYYYMMDD-HH>/…
```

- `<agent_code>`：绑定的智能体代码（不是岗位显示名）
- `<YYYYMMDD-HH>`：UTC 小时桶，避免互相覆盖
- 对话里可用 `@@docs/roles/…/某文件.md@@` 引用

约定说明见仓库 `docs/roles/README.md`（产出知识库根）。稳定的操作说明应沉淀进 `docs/user/`，不要把值班产出当产品文档 SSOT。

## 和其它能力怎么选

| 场景 | 更合适 |
|------|--------|
| 跟人多轮聊、临时探索 | 实时聊天 + 切换智能体 |
| 一次大型多角色软件交付 | [Plan 模式](../chat/plan-mode.md) / [任务中心](../tasks/task-center.md) |
| 到点重复跑固定 prompt | [自动化](../tasks/scheduled-tasks.md) |
| 长期盯仓库/项目、主动巡检与交班 | **智能体员工**（本文） |

值班路径默认不走聊天里的 Supervisor 大图拆派；员工在自己的岗位会话里干活（`proactive:{agent_code}`）。

## 常见问题

**Q：侧栏没有「智能体员工」？**  
确认客户端版本包含该入口；入口在聊天壳侧栏，文案为「智能体员工」。若提示 Premium / 授权相关，按许可证说明处理。

**Q：雇佣时选不到工作空间？**  
先在聊天侧栏添加本机工作区目录，再回来雇佣。

**Q：心跳在跳但事项全是失败 / 交班未完成？**  
说明岗在跑、交班没闭环。打开该员工工作轨迹看最后一轮工具与小结；优先查超时、上班时间窗外、工作区权限，而不是先改职责文案。

**Q：和「自动化」有何区别？**  
自动化是按 Cron 跑固定任务提示词；智能体员工是编内岗位：有职责、看板、审批、组织关系与交班文档。

**Q：改了智能体的 SOUL，员工会变吗？**  
会。岗位绑定的是同一 `agent_code`；人设与工具以「智能体」页为准，岗位页只管职责/节奏/工作区/审批。

**Q：归档后历史还在吗？**  
在。归档岗不自动上班、不在名册；可在「归档」Tab 重新上岗。彻底删除岗位不会删除「智能体」页里的 Agent 定义。
