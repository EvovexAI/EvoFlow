# 智能体（角色）管理

> **比如你可以这样创建角色**
>
> 在聊天里直接说：
>
> "帮我创建一个专门写 Python 数据分析脚本的智能体，偏好 Pandas + Polars，注释详细，默认用 DeepSeek V4"
>
> AI 会自动生成这个角色的 SOUL（人设）、系统提示词、工具白名单，保存即用。之后随时在顶栏切换到它，让它专门干活。
>
> 每个角色就像你请的一个"专家"——有自己的性格、擅长的工具、能调用的技能。角色可以被[智能体员工](smart-employees.md)雇佣上岗，实现定时巡检与主动工作。
>
> 读完本文你会学会三种创建角色的方式（对话、表单、TOML）、配置项含义、以及切换和管理方法。

智能体（Agent，又称"角色"）是 EvoFlow 的核心概念之一——每个角色绑定一套**人设（SOUL）、模型、技能、工具、MCP 与权限白名单**，相当于一个被精细调教过的专家。本文介绍三种创建方式、配置项含义、切换与管理流程。

---

## 三种创建方式

| 方式 | 适合 | 操作 |
|------|------|------|
| **对话直接触发** | 想法已成形，要快速试 | 在聊天里直接说"帮我创建一个 XX 角色"，主智能体自动生成 |
| **GUI 表单创建** | 需要精细控制权限与提示词 | 左侧菜单「智能体」→「新建智能体」按表单填写 |
| **直接编辑 TOML** | 批量管理 / 跨实例同步 | 编辑 `~/.evoflow/agents/<code>.toml`，重启或热重载生效 |

三种方式落到磁盘上是**同一份 TOML**，可互相覆盖。

---

## 对话快速创建

最低门槛的创建方式——把需求一次性说清楚：

```
帮我创建一个专门写 Python 数据分析脚本的智能体：
- 偏好 Pandas + Polars，少用 Numpy 直接索引
- 注释详细，每段代码前先说明意图
- 默认使用 DeepSeek V4 模型
```

主智能体会：

1. 生成 `agent_code`（如 `python-data-analyst`）
2. 写好 SOUL（人设）与系统提示词
3. 勾选合理的工具子集（read、write、terminal 等）
4. 落盘 TOML 并热重载

完成后顶栏出现新角色，立即可用。

## GUI 表单字段说明

进入「智能体」→「新建智能体」，按以下字段填写：

### 基础信息

| 字段 | 必填 | 说明 |
|------|------|------|
| **角色名称** | 是 | 显示用，如「Python 数据分析师」 |
| **agent_code** | 是 | 唯一英文标识，作为 TOML 文件名 |
| **头像 / 图标** | 否 | 可选上传或选用预设 emoji |
| **简短描述** | 推荐 | 一句话定位，出现在角色选择器里 |

### SOUL（人设）

定义角色的**人物画像**：是谁、说话风格、价值观、决策偏好。SOUL 偏"软"（人格），系统提示词偏"硬"（执行规则），二者分开维护。

### 系统提示词

直接拼到 LLM 的 system message，定义工作流程、产出格式、禁止事项等硬约束。建议结构：

```
# 角色
你是一名 ...

# 工作流程
1. 接到需求先确认 ...
2. 然后 ...

# 输出规范
- 代码块标语言
- 每段先说意图

# 禁止
- 不要 ...
```

### 模型绑定

| 字段 | 说明 |
|------|------|
| **主模型** | 该角色默认使用的对话模型 |
| **思考预算** | 支持扩展思考的模型可设上限（如 `thinking_budget: 8000`） |
| **温度 / Top-P** | 默认跟随面板设置，可单独覆盖 |

### 能力白名单

权限**按角色独立**，遵循最小权限原则：

| 类别 | 控制项 |
|------|--------|
| **技能** | 勾选该角色可调用的 `SKILL.md` 技能；未勾选的技能在该角色对话时**不会出现** |
| **工具** | 勾选可用内置工具（read、terminal、web_search 等）；高风险工具如 `delete`、`terminal` 建议谨慎授予 |
| **MCP** | 勾选可访问的 MCP 服务器；未勾选则该角色无法看到对应外部能力 |
| **会话模式** | 限定可用模式（Chat / Plan / Execute / Hosted） |

### 项目团队（可选）

将多个 `project-*` 角色组合成**项目团队**，可被 Plan 模式作为一个调度单元派发。详见 [项目级 Plan 流程](../chat/project-team-plan-workflow.md)。

## 切换使用

| 入口 | 操作 |
|------|------|
| **顶栏下拉** | 聊天页顶部「角色」按钮，选择目标角色 |
| **自然语言** | 在聊天里说"切到 Python 数据分析师"，主智能体自动切换 |
| **快捷键** | 在「设置 → 快捷键」绑定切换组合键 |
| **快捷指令** | `/agent <code>`（如 `/agent python-data-analyst`） |

切换后**当前会话历史保留**，但下一轮起 system message 与可用工具会被替换为目标角色。

---

## 管理已有角色

进入「智能体」页面（侧栏文案为「智能体」，路由 `#/expert`）：

- **编辑**：修改 SOUL、提示词、模型、白名单；保存后**热重载，无需重启**
- **复制**：基于现有角色快速派生变体（适合做 A/B 调参）
- **雇佣 / 值班台**：把该角色编入 [智能体员工](smart-employees.md) 岗位（定时上岗、看板与审批）
- **导出**：下载该角色的 TOML，便于跨实例同步或备份
- **删除**：移除不再用的角色（**内置 `project-*` 系列角色不可删除**，只能禁用）

## 与「智能体员工」的关系

| | **智能体**（本文） | **智能体员工** |
|--|-------------------|----------------|
| 侧栏 | 智能体 | 智能体员工 |
| 本质 | 能力包（SOUL / 工具 / 技能） | 岗位合同（职责 / 工作区 / 心跳 / 审批） |
| 触发 | 聊天里 @ 或切换角色 | 定时心跳、派发、立即开工 |

要让某个角色「像员工一样主动巡检」，先按本文建好 Agent，再去 [智能体员工](smart-employees.md) **雇佣**。

---

## 角色与项目、记忆的关系

| 维度 | 默认行为 |
|------|---------|
| **角色与会话** | 一个会话当前只激活一个主角色（Plan 模式下子任务可临时切到其他角色） |
| **角色与记忆** | 每个角色拥有**独立记忆文件**，互不污染；可在「记忆管理」分 Tab 编辑 |
| **角色与项目** | 项目可指定**默认角色**；切到该项目时自动套用 |
