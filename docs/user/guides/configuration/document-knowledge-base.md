# 上传文档（RAG 知识库）

> **比如你有一批产品手册、技术规范 PDF，想让 AI 在对话中引用**：
>
> 1. 在"上传文档"页面创建一个知识库，绑定向量模型
> 2. 上传所有 PDF 文件，AI 自动分块和摘要
> 3. 在聊天里问"我们产品的最大并发连接数是多少？"——AI 从知识库里检索到答案
>
> 适合把零散文档转化为 AI 可检索的知识。跟[知识库（Obsidian Vault）](knowledge-vault.md)的区别是：这里上传的文档做向量检索，Vault 连接本地笔记目录保留原文件。

把 PDF / Word / Markdown 等资料做成**可向量检索的知识库**，对话里由工具 `search_knowledge_base` 引用。侧栏文案：**上传文档**（路由 `#/knowledge`）。

> **默认侧栏可能不显示本入口**（避免与「知识库」并排两个菜单）。仍可直接打开 `#/knowledge`，或从「知识库」页里的相关跳转进入。Obsidian / 本地笔记库请用侧栏 **[知识库](knowledge-vault.md)**（`#/knowledge/vaults`），不要和本页混淆。

---

## 和其它「知识」有何不同

| 能力 | 入口 | 存什么 | Agent 怎么用 |
|------|------|--------|--------------|
| **上传文档**（本文） | `#/knowledge` | 上传文件 → EvoFlow 向量库 | `search_knowledge_base` |
| **知识库**（Vault） | `#/knowledge/vaults` | 本地 Obsidian/Markdown 目录 | `knowledge(action=…)` |
| **记忆文件** | `#/memory` | `memory.json` 用户事实/偏好 | 对话自动注入 / 回忆 |
| **会话文件上传** | 聊天 Composer | 当前线程附件 | 单次会话解析，**不是**长期知识库 |

详见 [文件上传](../chat/file-upload.md)（会话级）、[记忆管理](memory-management.md)、[知识库](knowledge-vault.md)。

## 前置条件

1. 已配置对话模型
2. 在 **设置 → 模型 → 向量模型** 中至少添加一个可用向量模型（新建知识库时必选）
3. （可选）本机有待扫描的文档文件夹

---

## 快速开始

1. 打开 `#/knowledge`（或侧栏「上传文档」若已显示）
2. 点 **＋ 新建知识库**
3. 填写 **名称**、选择 **向量模型**；可选勾选 **LLM 智能索引**、绑定 **本地文件夹**
4. 进入详情 → **文档** Tab → **上传文档** / **上传文件夹**
5. 等状态变为 **已就绪** 后，到 **检索测试** 试搜几句
6. 在聊天中让 Agent「根据知识库回答…」（依赖角色已授权 `search_knowledge_base`）

## 新建知识库

| 字段 | 必填 | 说明 |
|------|------|------|
| **名称** | 是 | 如：产品文档 |
| **向量模型** | 是 | 无选项时先去设置添加向量模型 |
| **本地文件夹** | 否 | 绑定后可自动扫描；桌面端可「选择文件夹…」 |
| **LLM 智能索引** | 否 | 默认开：语义分块+摘要；关则规则分块 |

## 详情页三个 Tab

### 文档

- 左侧 **目录** 树；可新建目录
- **上传文档**：常见 `.txt` / `.md` / `.pdf` / `.docx` / `.pptx` / `.xlsx` / `.csv` / `.json` 等
- **上传文件夹**：整目录导入
- 可按目录管理、删除单篇文档

### 检索测试

| 控件 | 说明 |
|------|------|
| 检索文本 | 查询语句 |
| **Top-K** | 1–20，默认 5 |
| **检索** | 查看命中片段，验收索引是否可用 |

### 设置

| 项 | 说明 |
|----|------|
| 名称 / 描述 | 基本信息 |
| 文件夹路径 | 留空则仅手动上传 |
| **选择文件夹…** / **重新扫描** / **强制全量重建** | 本地目录同步与重建索引 |
| **LLM 智能索引** | 同创建时 |
| **保存** | 保存配置 |
| **删除知识库** | 危险操作，不可恢复 |

## 常见问题

**Q：侧栏没有「上传文档」？**  
属默认隐藏。用地址栏进入 `#/knowledge`，或从知识库页跳转。开发者可将 `SHOW_KNOWLEDGE_NAV` 设为可见（见 `evopanel/src/lib/nav-visibility.js`）。

**Q：新建时提示没有向量模型？**  
到 **设置 → 模型 → 向量模型** 添加后再建库。

**Q：和聊天里拖文件有何区别？**  
聊天上传只服务**当前会话**；本页是**长期可检索**的 RAG 库，多会话可复用。

**Q：Agent 搜不到内容？**  
1）知识库状态是否「已就绪」；2）检索测试是否有命中；3）当前智能体是否授权 `search_knowledge_base`。
