# 案例八：Supervisor 模式（超级总控智能体）

## 场景说明

当你面对一个复杂任务，需要多个专业角色协同完成时，Supervisor（超级总控智能体）模式就派上用场了。超级总控智能体会自动拆解任务、分配给多个子智能体、监督执行进度，并在出错时自动调整方案。

## 前置准备

- 已完成 [安装](../getting-started/installation.md)
- 已配置至少一个 AI 模型
- 已创建至少一个自定义智能体（作为子智能体候选）

## 操作步骤

### 步骤 1：打开对话并进入规划模式


<!-- 截图占位：supervisor-step1.png —  -->

> **截图说明**：展示对话模式选择，标注"规划"模式。

### 步骤 2：输入复杂任务

向智能体描述一个需要多步骤、多角色协作的复杂任务。

**示例提示词：**

```
我想开发一个企业内部的技术日报系统，需求如下：
1. 每天自动搜集技术新闻（AI、云计算、开源领域）
2. 将新闻按技术领域分类整理
3. 生成 HTML 格式的日报
4. 通过飞书推送到技术团队群组
5. 提供历史日报查询功能

请帮我规划实现方案。
```

<!-- 截图占位：supervisor-step2.png — 输入复杂任务 -->

> **截图说明**：展示输入框中输入复杂任务的界面。

### 步骤 3：超级总控智能体澄清需求

超级总控智能体会进入澄清阶段，通过提问确认任务范围、非目标和验收标准。

**超级总控智能体可能提出的问题：**

- 技术日报需要覆盖哪些具体的技术方向？
- 推送时间是每天几点？
- 历史日报需要保留多久？
- 是否需要支持多个团队的不同日报？

<!-- 截图占位：supervisor-step3.png — 澄清需求 -->

> **截图说明**：展示超级总控智能体提问澄清需求的对话界面。

### 步骤 4：任务分解

澄清完成后，超级总控智能体会将任务拆解为多个子任务，并规划执行顺序和依赖关系。

**示例分解结果：**

| 子任务 | 负责角色 | 依赖 |
|--------|----------|------|
| 新闻搜集技能配置 | 爬虫专家 | 无 |
| 新闻分类逻辑开发 | 数据工程师 | 新闻搜集 |
| HTML 模板设计 | 前端开发 | 无 |
| 飞书推送配置 | 运维工程师 | HTML 模板 |
| 自动化编排 | 系统集成 | 以上全部 |

<!-- 截图占位：supervisor-step4.png — 任务分解 -->

> **截图说明**：展示超级总控智能体分解的子任务列表和依赖关系图。

### 步骤 5：分配子智能体

超级总控智能体会根据子任务的性质，选择合适的智能体来执行：

- 主智能体自身处理通用任务
- 自定义智能体（如"数据分析助手"）处理专业任务
- Claude Code 处理编码任务

<!-- 截图占位：supervisor-step5.png — 分配子智能体 -->

> **截图说明**：展示超级总控智能体分配子任务给不同智能体的界面。

### 步骤 6：监督执行

超级总控智能体会实时监控各子任务的执行进度，处理错误和重试。你可以在对话中看到整体进度和各子任务的状态。

<!-- 截图占位：supervisor-step6.png — 监督执行 -->

> **截图说明**：展示超级总控智能体监督界面，标注各子任务的执行状态（进行中/已完成/失败）。

### 步骤 7：处理异常

如果某个子任务失败，超级总控智能体会自动分析原因，尝试重新分配或调整方案。

**示例场景：**

```
⚠️ 子任务"新闻分类逻辑开发"执行失败：
  原因：数据格式与预期不符
  处理：重新调整分类规则，再次执行
```

<!-- 截图占位：supervisor-step7.png — 处理异常 -->

> **截图说明**：展示超级总控智能体处理子任务失败的界面，标注自动重试的过程。

### 步骤 8：任务完成与总结

所有子任务完成后，超级总控智能体会汇总结果，向你报告整体完成情况。

<!-- 截图占位：supervisor-step8.png — 任务完成 -->

> **截图说明**：展示超级总控智能体汇总报告界面，标注所有子任务的完成状态。

## 小结

超级总控智能体模式是 EvoFlow 处理复杂任务的核心能力。它将"一个人干所有事"变为"一个团队协同作战"，大幅提升复杂项目的交付效率和质量。
