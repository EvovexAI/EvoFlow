# 案例二：创建智能体

## 场景说明

主智能体适合通用任务，但如果你有特定领域的需求（如专门做数据分析、内容审核、代码审查），可以创建一个具有专属身份和能力的自定义智能体。

## 前置准备

- 已完成 [安装](../getting-started/installation.md)
- 已配置至少一个 AI 模型
- 了解 SOUL.md 的基本概念（智能体的"人格"定义）

## 操作步骤

### 步骤 1：进入角色管理

在 EvoPanel 左侧导航栏中，点击 **「角色」** 图标，进入智能体管理页面。

<!-- 截图占位：create-agent-step1.png — 进入角色管理 -->

> **截图说明**：展示左侧导航栏，标注"角色"图标位置。

### 步骤 2：点击创建新智能体

在角色管理页面右上角，点击 **「+ 新建智能体」** 按钮。

<!-- 截图占位：create-agent-step2.png — 点击新建 -->

> **截图说明**：展示角色管理页面，标注"新建智能体"按钮。

### 步骤 3：填写基本信息

在弹出的表单中填写智能体的基本信息：

- **名称**：给智能体起一个名字（如"数据分析助手"）
- **描述**：简要说明这个智能体的用途

<!-- 截图占位：create-agent-step3.png — 填写基本信息 -->

> **截图说明**：展示新建智能体表单，标注名称和描述输入框。

### 步骤 4：编写 SOUL.md（身份定义）

SOUL.md 是智能体的核心身份文件，用自然语言描述这个智能体的：

- 性格和行为准则
- 专业领域和擅长什么
- 沟通风格（正式/随意/简洁/详细）
- 自主程度（是否需要用户确认再行动）

**示例 SOUL.md 提示词（你可以直接告诉智能体帮你写）：**

在对话中向主智能体输入以下指令，让它帮你生成 SOUL.md：

```
请帮我创建一个数据分析智能体的 SOUL.md，要求：
- 专业、严谨的数据分析专家
- 擅长用 Python 做数据清洗、统计分析和可视化
- 回答时先理解需求，再给出分析方案
- 代码要附注释，结果用图表或表格呈现
- 不确定时会主动询问澄清
```

主智能体会生成一份 SOUL.md 内容，你复制粘贴到智能体编辑页面的 SOUL.md 区域。

<!-- 截图占位：create-agent-step4.png — 编写 SOUL.md -->

> **截图说明**：展示 SOUL.md 编辑区域，标注输入框和内容示例。

### 步骤 5：配置模型和工作区

- **模型**：为该智能体选择一个专用的 AI 模型（可以与主智能体不同）
- **工作区**：选择或创建一个文件夹作为该智能体的工作目录

<!-- 截图占位：create-agent-step5.png — 配置模型和工作区 -->

> **截图说明**：展示模型选择下拉框和工作区路径选择界面。

### 步骤 6：添加工具和技能

选择该智能体可以使用的工具和已安装的技能。比如数据分析智能体可以添加：

- 工具：文件读写、命令执行、端口检测
- 技能：data-analysis、summarize

<!-- 截图占位：create-agent-step6.png — 添加工具和技能 -->

> **截图说明**：展示工具和技能选择列表，标注已勾选的项目。

### 步骤 7：保存并测试

点击 **「保存」**，然后在角色列表中找到新建的智能体，点击 **「对话」** 开始测试。

**测试提示词：**

```
你好，请帮我分析这个 CSV 文件中的数据趋势。
```

<!-- 截图占位：create-agent-step7.png — 保存并测试 -->

> **截图说明**：展示保存后的智能体列表，标注对话入口。

## 小结

创建智能体后，你可以在对话中切换到这个智能体，或者通过自动化、IM 渠道让它自动执行任务。每个智能体拥有独立的记忆、模型配置和能力集。
